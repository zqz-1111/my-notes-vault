# 苍穹外卖微服务拆分：从单体到微服务的完整实战

## 一、写在前面

### 为什么要做微服务拆分？

苍穹外卖原本是一个**单体架构**项目，所有模块（用户、商品、订单、支付、文件上传）都打包在一个 Spring Boot 应用里。随着业务增长，单体架构会面临：

- **部署成本高**：改一行代码就要重新部署整个应用
- **扩展性差**：只能整体扩展，不能按模块单独扩容
- **技术债务**：模块间耦合紧密，改一个功能可能影响其他模块

微服务拆分后，每个模块独立部署、独立扩展、独立数据库，通过 HTTP（OpenFeign）进行服务间通信。

### 学完这篇你能掌握什么

1. 如何把一个单体 Spring Boot 项目拆成多个微服务
2. Spring Cloud 核心组件的实战用法：Nacos、Gateway、OpenFeign
3. 服务间通信的完整链路设计
4. 微服务拆分中常见的坑和解决方案

```

├── user-service      ← 用户服务（端口 8081）

|------|------|------------------|--------------|---------|

| file-service | 8086 | CommonController | **无数据库** | MinIO |

### 请求链路全景图

```
前端（浏览器）
    ↓
Nginx（端口 80）
    ↓ /api/* → http://localhost:8080/admin/*
sky-gateway（端口 8080）
    ├── 白名单放行（登录、店铺状态等）
    ├── 解析 JWT Token → 提取 userId/empId
    ├── 请求头塞入 user-info
    ├── 路由转发到对应微服务
    └── 无 Token → 返回 401
    ↓
各微服务（8081~8086）
    ├── UserInfoInterceptor 从请求头读取 user-info → 存入 ThreadLocal
    ├── 处理业务逻辑
    └── 需要跨服务调用 → 通过 OpenFeign + Nacos 服务发现
```

    <artifactId>spring-cloud-alibaba-dependencies</artifactId>
    <version>${spring-cloud-alibaba.version}</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>
```

3. `<modules>` 中注册 8 个新模块（sky-api、sky-gateway、user/product/trade/data/pay/file-service）

---

### 第 2 步：修改 sky-common 公共模块

**目的**：把各服务共用的组件抽到公共模块。

**关键改动**：

1. **新增 `UserInfoInterceptor`** — 从 Gateway 传递的请求头 `user-info` 中获取用户 ID，存入 ThreadLocal
```java
public class UserInfoInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, ...) {
        String userId = request.getHeader("user-info");
        if (userId != null) {
            BaseContext.setCurrentId(Long.valueOf(userId));
        }
        return true;
    }
    @Override
    public void afterCompletion(...) {
        BaseContext.removeCurrentId(); // 防止内存泄漏
    }
}
```

2. **迁移 `AutoFill` 注解和 `AutoFillAspect` 切面** — 从 sky-server 移到 sky-common，否则各微服务的 Mapper 无法自动填充 createTime/updateTime

3. **添加 `spring-boot-starter-web` 依赖**（scope: provided）— 编译时需要，运行时由各微服务自己提供

---

### 第 3 步：创建 sky-api 模块（Feign 客户端）

**目的**：统一定义所有跨服务调用的接口，被所有业务微服务依赖。

**核心组件**：

#### DefaultFeignConfig — Feign 拦截器
每次 Feign 调用时自动将当前用户 ID 传递给下游服务：
```java
public class DefaultFeignConfig {
    @Bean
    public RequestInterceptor userInfoRequestInterceptor() {
        return template -> {
            Long currentId = BaseContext.getCurrentId();
            if (currentId != null) {
                template.header("user-info", currentId.toString());
            }
        };
    }
}
```

#### 四个 Feign 客户端接口

| FeignClient | 调用目标 | 被谁调用 | 主要方法 |
|-------------|---------|---------|---------|
| `ProductClient` | product-service | trade-service、data-service | getById、countDishByMap、countSetmealByMap |
| `UserClient` | user-service | trade-service、data-service | getAddressBookById、countUserByMap |
| `TradeClient` | trade-service | data-service、pay-service | sumOrderByMap、countOrderByMap、getSalesTop10、markOrderPaySuccess |
| `FileClient` | file-service | product-service | upload |

**关键点**：每个 FeignClient 都要配置 `configuration = DefaultFeignConfig.class`，否则用户信息不会传递。

### 第 10 步：创建 sky-gateway（端口 8080）

**职责**：统一入口、路由转发、JWT 鉴权、跨域处理。

**核心组件**：

#### AuthGlobalFilter — 全局鉴权过滤器
```java
@Override
public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
    String path = request.getPath().toString();
    
    // 1. 白名单放行
    if (isExclude(path)) return chain.filter(exchange);
    
    // 2. 尝试管理端 Token（请求头: token）
    String adminToken = request.getHeaders().getFirst("token");
    Claims claims = Jwts.parser()
        .setSigningKey(secretKey.getBytes(StandardCharsets.UTF_8))  // ⚠️ 必须用 UTF-8 字节
        .parseClaimsJws(adminToken).getBody();
    Long empId = Long.valueOf(claims.get("empId").toString());
    
    // 3. 请求头塞入 user-info，传递给下游服务
    ServerHttpRequest newRequest = request.mutate()
        .header("user-info", empId.toString()).build();
    return chain.filter(exchange.mutate().request(newRequest).build());
}
```

#### 路由配置（application.yml）
```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user
          uri: lb://user-service
          predicates:
            - Path=/admin/employee/**,/user/user/**,/user/addressBook/**
        - id: product
          uri: lb://product-service
          predicates:
            - Path=/admin/category/**,/admin/dish/**,/admin/setmeal/**,...
        - id: trade
          uri: lb://trade-service
          predicates:
            - Path=/admin/order/**,/user/order/**,/user/shoppingCart/**
        - id: data
          uri: lb://data-service
          predicates:
            - Path=/admin/report/**,/admin/workspace/**
        - id: pay
          uri: lb://pay-service
          predicates:
            - Path=/pay/**
        - id: file
          uri: lb://file-service
          predicates:
            - Path=/admin/common/**
        - id: trade-ws
          uri: lb:ws://trade-service        # WebSocket 用 ws:// 协议
          predicates:
            - Path=/ws/**
```

**重要**：Gateway 基于 WebFlux，**不能引入 spring-boot-starter-web**！

## 五、模块间通信机制详解

### 通信方式：OpenFeign + Nacos 服务发现

```
服务A（调用方）
    ↓ 注入 FeignClient
    ↓ 调用方法（如 productClient.getById(1L)）
    ↓
OpenFeign 框架
    ↓ 根据 @FeignClient(value = "product-service") 找到服务名
    ↓
Nacos 服务注册中心
    ↓ 返回 product-service 的实例列表（如 192.168.x.x:8082）
    ↓
负载均衡（LoadBalancer）
    ↓ 选择一个实例
    ↓
发送 HTTP 请求到目标服务
```

### 用户信息传递链路

```
前端请求（带 Token）
    ↓
Nginx 转发到 Gateway
    ↓
Gateway AuthGlobalFilter
    ├── 解析 JWT → 提取 empId/userId
    └── 请求头塞入 user-info
    ↓
下游微服务
    ├── UserInfoInterceptor 从请求头读取 user-info → 存入 ThreadLocal
    └── 业务代码通过 BaseContext.getCurrentId() 获取用户 ID
    ↓
如果需要继续调用其他服务（Feign）
    └── DefaultFeignConfig 从 ThreadLocal 读取 → 塞入 Feign 请求头
```

这条链路保证了**用户信息在整条调用链中不断裂**。

## 六、踩坑避坑指南

### 坑 1：MinIOUtil Bean 找不到

**报错**：
```
NoSuchBeanDefinitionException: No qualifying bean of type 'com.sky.utils.MinIOUtil'
```

**原因**：MinIOUtil 使用 `@AllArgsConstructor` 但没有 `@Component`，不是 Spring Bean，需要手动注册。

**解决方案**：在各服务的 WebMvcConfiguration 中手动注册 Bean：
```java
@Bean
public MinIOUtil minIOUtil(MinIOProperties minIOProperties) {
    return new MinIOUtil(
        minIOProperties.getEndpoint(),
        minIOProperties.getAccessKey(),
        minIOProperties.getSecretKey(),
        minIOProperties.getBucketName()
    );
}
```

**影响范围**：file-service、product-service（DishServiceImpl 注入了 MinIOUtil）

---

### 坑 2：PageHelper 循环依赖

**报错**：
```
The dependencies of some of the beans form a cycle: PageHelperAutoConfiguration
```

**原因**：`pagehelper-spring-boot-starter:1.3.0` 和 Spring Boot 2.7.x 有兼容性 bug。

**解决方案**：根 pom.xml 升级版本：
```xml
<pagehelper>1.4.6</pagehelper>  <!-- 从 1.3.0 升级 -->
```

---

### 坑 3：Knife4j（Springfox）NPE

**报错**：
```
Failed to start bean 'documentationPluginsBootstrapper': NullPointerException
```

**原因**：Spring Boot 2.7.x 默认路径匹配策略改成 `path-pattern-parser`，但 Springfox 3.0.0 只支持 `ant-path-matcher`。

**解决方案**：所有用了 Knife4j 的服务的 application.yml 加：
```yaml
spring:
  mvc:
    pathmatch:
      matching-strategy: ant_path_matcher
```

---

### 坑 4：JWT 配置缺失导致登录 500

**报错**：
```
NullPointerException at JwtUtil.createJWT
```

**原因**：user-service 的 application.yml 没有配置 `sky.jwt.*`，导致 JwtProperties 的字段全是 null。

**解决方案**：user-service 的 yml 必须和 Gateway 完全一致的 JWT 配置：
```yaml
sky:
  jwt:
    admin-secret-key: itcast
    admin-ttl: 7200000
    admin-token-name: token
    user-secret-key: itheima
    user-ttl: 7200000
    user-token-name: authentication
```

---

### 坑 5：登录后全部 401（JWT 签名方式不匹配）

**现象**：登录成功返回 Token，但后续请求全部 401。

**原因**：jjwt 0.9.1 的 `setSigningKey(String)` 做 **Base64 解码**，但 `JwtUtil.createJWT` 的 `signWith(SignatureAlgorithm, byte[])` 用的是 **UTF-8 原始字节**。两边编码方式不同，Token 永远校验失败。

**解决方案**：Gateway 的 AuthGlobalFilter 中，解析 Token 时必须用 UTF-8 字节：
```java
// ❌ 错误：setSigningKey(String) 会做 Base64 解码
.setSigningKey(jwtProperties.getAdminSecretKey())

// ✅ 正确：手动转 UTF-8 字节，和签名方式一致
.setSigningKey(jwtProperties.getAdminSecretKey().getBytes(StandardCharsets.UTF_8))
```

**这是最容易忽略的坑**，因为登录接口在白名单里不走鉴权，所以登录正常，但所有其他接口都 401。

---

### 坑 6：Feign 调用 404（Controller 路径重复）

**报错**：
```
FeignException$NotFound: [404] during [GET] to [http://trade-service/admin/order/sumByMap]
```

**原因**：Controller 类上有 `@RequestMapping("/admin/order")`，方法上又写了 `@GetMapping("/admin/order/sumByMap")`，导致实际路径变成 `/admin/order/admin/order/sumByMap`（重复）。

**解决方案**：方法上的 @GetMapping 只写相对路径：
```java
@RequestMapping("/admin/order")  // 类上
public class OrderController {
    @GetMapping("/sumByMap")     // ✅ 实际路径 /admin/order/sumByMap
    public Double sumByMap(...) { ... }
}
```

---

### 坑 7：FeignClient 路径和实际接口不匹配

**报错**：
```
FeignException$NotFound: [404] during [GET] to [http://user-service/admin/user/countByMap]
```

**原因**：FeignClient 写的是 `/admin/user/countByMap`，但接口实际在 `EmployeeController` 里，路径是 `/admin/employee/countByMap`。

**解决方案**：FeignClient 的路径必须和目标 Controller 的实际路径一致：
```java
// ❌ 错误
@GetMapping("/admin/user/countByMap")

// ✅ 正确（接口在 EmployeeController 里）
@GetMapping("/admin/employee/countByMap")
```

---

### 坑 8：WebSocket 连接失败

**现象**：WebSocket 连接 `ws://localhost/ws/xxx` 失败。

**原因**：Gateway 没有 WebSocket 路由配置，Nginx 转发到 Gateway 后找不到目标。

**解决方案**：

1. Gateway 的 application.yml 添加 WebSocket 路由：
```yaml
- id: trade-ws
  uri: lb:ws://trade-service  # 注意用 ws:// 协议
  predicates:
    - Path=/ws/**
```

2. 白名单中加 `/ws/**`（WebSocket 连接不走 Token 校验）

---

### 坑 9：Gateway 不能引入 spring-boot-starter-web

**报错**：
```
A component required a bean of type 'ServerCodecConfigurer'
```

**原因**：Gateway 基于 WebFlux，和 spring-boot-starter-web（Servlet）冲突。

**解决方案**：Gateway 的 pom.xml 绝对不能有 spring-boot-starter-web 依赖。

- ✅ 工作台数据概览（Gateway → data-service → Feign 调用 trade/user/product 三个服务）
- ✅ WebSocket 来单提醒（Gateway → trade-service WebSocket）
- ✅ 文件上传（Gateway → file-service → MinIO）

## 八、全文总结

### 核心知识点

1. **微服务拆分原则**：高内聚低耦合，纵向按功能模块拆分
2. **Nacos 服务注册发现**：各服务启动时自动注册，Feign 通过服务名找到实例
3. **OpenFeign 声明式调用**：接口 + 注解 → 动态代理 → HTTP 请求
4. **Gateway 统一鉴权**：一个过滤器搞定所有服务的 Token 校验，下游服务只需读请求头
5. **用户信息透传**：Gateway 塞请求头 → 微服务拦截器读取 → ThreadLocal → Feign 拦截器转发

### 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 鉴权位置 | Gateway 统一鉴权 | 各服务不再需要 JWT 拦截器，代码更简洁 |
| 服务间通信 | OpenFeign | 声明式、和 Spring Boot 集成好、支持负载均衡 |
| 注册中心 | Nacos | 阿里开源、支持配置管理和服务发现 |
| 用户信息传递 | 请求头 user-info | 简单可靠，全链路自动传递 |

### 模块依赖关系

```
sky-common ← sky-pojo ← sky-api
                        ↑
          ┌─────────────┼─────────────┐
          │             │             │
    user-service  product-service  trade-service
          │             │             │
          └─────────────┼─────────────┘
                        ↓
                  data-service（纯聚合，无数据库）
                  pay-service（支付，无数据库）
                  file-service（文件，无数据库）
                        ↓
                  sky-gateway（统一入口）
```
