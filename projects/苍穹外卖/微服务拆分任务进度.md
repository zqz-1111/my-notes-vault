# 微服务拆分任务进度

## 一、作业目标

将苍穹外卖项目拆分为微服务项目，完成以下需求：

1. **基于 OpenFeign 实现服务间远程调用**
2. **定义网关，实现对微服务的请求路由**
3. **基于网关实现登录用户校验和用户信息传递**

## 三、核心架构

### 3.1 用户信息传递链路

```
前端请求 → Gateway(JWT鉴权,解析userId) 
         → 添加 user-info 请求头 
         → 微服务(UserInfoInterceptor从请求头取userId存入ThreadLocal)
         → Feign调用(Feign拦截器从ThreadLocal取userId传递给下游)
```

### 3.2 跨服务调用链路

```
trade-service
├── ShoppingCartServiceImpl → ProductClient.getById()        → product-service
├── ShoppingCartServiceImpl → ProductClient.getSetmealById() → product-service
└── OrderServiceImpl       → UserClient.getAddressBookById() → user-service

data-service
├── WorkspaceServiceImpl → TradeClient.sumOrderByMap()       → trade-service
├── WorkspaceServiceImpl → TradeClient.countOrderByMap()     → trade-service
├── WorkspaceServiceImpl → UserClient.countUserByMap()       → user-service
├── WorkspaceServiceImpl → ProductClient.countDishByMap()    → product-service
└── ReportServiceImpl    → TradeClient.getSalesTop10()       → trade-service

pay-service
└── PayServiceImpl → TradeClient.markOrderPaySuccess()       → trade-service
```

## 五、参考资料

### 5.1 黑马商城拆分经验

已学习的黑马商城微服务知识点：
- **Gateway 鉴权过滤器**：AuthGlobalFilter 白名单判断 + Token 校验 + userId 传递到请求头
- **微服务获取用户**：UserInfoInterceptor 从请求头读取 → 存入 UserContext ThreadLocal
- **OpenFeign 传递用户**：DefaultFeignConfig 中 RequestInterceptor 实现用户信息传递
- **服务注册发现**：Nacos 配置、心跳机制
- **微服务拆分原则**：高内聚低耦合、纵向拆分按功能模块

## 六、踩坑记录

### 6.1 sky-common 缺少 servlet 依赖

**问题**：UserInfoInterceptor 引用 `javax.servlet.http.HttpServletRequest` 爆红

**原因**：sky-common 没有 `spring-boot-starter-web` 依赖

**解决**：在 sky-common 的 pom.xml 添加依赖（provided scope）：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <scope>provided</scope>
</dependency>
```
