# 【登录功能】验证码发送实现

## 一、写在前面

1. 为什么学这个：登录是任何系统的基础功能，短信验证码登录是目前主流的登录方式之一
2. 学完这篇你能掌握：短信验证码的发送流程、手机号校验、Session 存储验证码

## 二、前置准备

- JDK 8 + Spring Boot 2.3.12
- Redis 已启动（localhost:6379）
- MySQL 已启动（localhost:3306，数据库 hmdp）
- 了解 Session 的基本概念

## 三、验证码发送流程

### 3.1 整体流程

```
前端发送手机号 → 后端校验 → 生成验证码 → 保存到 Session → 发送短信（模拟）
```

### 3.2 代码实现

**UserController.java**

```java
@PostMapping("code")
public Result sendCode(@RequestParam("phone") String phone, HttpSession session) {
    // 调用 service 层发送验证码
    return userService.sendCode(phone, session);
}
```

**UserServiceImpl.java**

```java
@Override
public Result sendCode(String phone, HttpSession session) {
    // 1. 校验手机号格式
    if (RegexUtils.isPhoneInvalid(phone)) {
        // 不符合格式，返回错误信息
        return Result.fail("手机号格式错误！");
    }
    
    // 2. 符合格式，生成6位随机验证码
    String code = RandomUtil.randomNumbers(6);
    
    // 3. 保存验证码到 Session
    session.setAttribute("code", code);
    
    // 4. 发送验证码（这里只是模拟，实际需要调用短信服务）
    log.debug("发送短信验证码成功，验证码：{}", code);
    
    // 5. 返回成功
    return Result.ok();
}
```

### 3.3 关键点说明

**手机号校验工具类**

```java
// RegexUtils.isPhoneInvalid(phone) 
// 使用正则表达式校验手机号格式
// 返回 true 表示格式无效，返回 false 表示格式有效
```

**验证码生成**

```java
// RandomUtil.randomNumbers(6)
// 使用 Hutool 工具生成6位纯数字验证码
// 生成结果如：123456
```

**Session 存储**

```java
session.setAttribute("code", code);
// 将验证码存入 HttpSession
// 后续登录时可以从 Session 中取出验证码进行比对
```

## 四、踩坑避坑指南

### 坑 1：JVM Target 版本不匹配

**报错信息：**
```
java: 无法编译为 JVM 目标 17 配置的模块 'hm-dianping': 指定的回退 SDK 版本 8 不支持所需 jvm 目标 17
```

**原因：** IDEA 的内部设置被改成了 target 17，但本地只有 JDK 8

**解决方案：** 在 IDEA 里改 3 个地方：
1. `File → Project Structure → Modules` → Language Level 改成 8
2. `File → Project Structure → Project` → Project SDK 和 Language Level 改成 8
3. `File → Settings → Build, Execution, Deployment → Compiler → Java Compiler` → Target bytecode version 改成 8

详细记录见：[[JVM Target 版本不匹配报错]]

## 五、效果验证

启动项目后，使用 Postman 或 curl 测试：

```bash
POST http://localhost:8081/user/code?phone=13800138000
```

预期结果：
- 手机号格式正确：返回 `{"success":true}`
- 手机号格式错误：返回 `{"success":false,"errorMsg":"手机号格式错误！"}`

控制台日志会输出：
```
DEBUG - 发送短信验证码成功，验证码：123456
```

## 六、全文总结

- 短信验证码发送的核心流程：校验 → 生成 → 存储 → 发送
- 使用 Session 存储验证码，后续登录时可以取出比对
- 手机号校验使用正则表达式，保证格式正确
- 实际项目中需要对接真实的短信服务（如阿里云、腾讯云）
