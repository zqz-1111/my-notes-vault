# 达人探店点赞功能改造

## 一、写在前面

1. 为什么学这个：原有点赞功能用 Redis Set 实现，只能记录"谁点过赞"，无法按时间排序。要实现"点赞排行榜"功能，需要换成 SortedSet
2. 学完你能掌握：Redis SortedSet 实现点赞/取消点赞 + 按时间排序的 Top5 排行榜

## 二、前置准备

- Redis SortedSet 基本命令：`ZADD`、`ZREM`、`ZSCORE`、`ZREVRANGE`
- 已有 Set 实现的点赞功能（见 [[秒杀功能实现与并发安全]]）

## 三、正文

### 3.1 问题分析

原有方案用 Set 存储点赞用户：
```java
// 判断是否点赞
SISMEMBER key userId
// 点赞
SADD key userId
// 取消点赞
SREM key userId
```

**Set 特点：** 元素唯一，但**无法排序**。

要实现"按点赞时间排序的排行榜"，需要换成 **SortedSet**：
- 元素唯一（同一用户只有一条记录）
- 按 score 排序（存时间戳，天然按时间排序）

### 3.2 改造 isBlogLiked() — 判断是否点赞

```java
private void isBlogLiked(Blog blog) {
    UserDTO user = UserHolder.getUser();
    if (Objects.isNull(user)){
        // 当前用户未登录，无需查询点赞
        return;
    }
    Long userId = user.getId();
    String key = BLOG_LIKED_KEY + blog.getId();
    // 改用 ZSCORE 替代 SISMEMBER
    Double score = stringRedisTemplate.opsForZSet().score(key, userId.toString());
    // score 不为 null 说明已点赞
    blog.setIsLike(Objects.nonNull(score));
}
```

**关键变化：** `SISMEMBER` → `ZSCORE`，判断 score 是否存在。

### 3.3 改造 likeBlog() — 点赞/取消点赞

```java
@Override
public Result likeBlog(Long id) {
    Long userId = UserHolder.getUser().getId();
    String key = BLOG_LIKED_KEY + id;
    
    // ZSCORE 判断是否已点赞
    Double score = stringRedisTemplate.opsForZSet().score(key, userId.toString());
    
    if (score == null) {
        // 未点赞 → 点赞：ZADD key score(userId时间戳) value(userId)
        result = this.update(...);  // liked + 1
        if (result) {
            stringRedisTemplate.opsForZSet().add(key, userId.toString(), System.currentTimeMillis());
        }
    } else {
        // 已点赞 → 取消：ZREM key value(userId)
        result = this.update(...);  // liked - 1
        if (result) {
            stringRedisTemplate.opsForZSet().remove(key, userId.toString());
        }
    }
    return Result.ok();
}
```

**关键变化：**
- `SADD` → `ZADD`，score 存 `System.currentTimeMillis()`
- `SREM` → `ZREM`

### 3.4 新增 queryBlogLikes() — 查询点赞排行榜

```java
@Override
public Result queryBlogLikes(Long id) {
    String key = BLOG_LIKED_KEY + id;
    // ZREVRANGE 按 score 倒序，取 Top5
    Set<String> top5 = stringRedisTemplate.opsForZSet().range(key, 0, 4);
    if (top5 == null || top5.isEmpty()) {
        return Result.ok(Collections.emptyList());
    }
    // 查询用户信息并转为 UserDTO
    List<Long> ids = top5.stream().map(Long::valueOf).collect(Collectors.toList());
    List<UserDTO> userDTOList = userService.listByIds(ids).stream()
            .map(user -> BeanUtil.copyProperties(user, UserDTO.class))
            .collect(Collectors.toList());
    return Result.ok(userDTOList);
}
```

**关键命令：** `ZRANGE key 0 4` 按 score 正序取前5，`ZREVRANGE key 0 4` 按 score 倒序取前5。

## 四、Redis 数据结构选型

| 特性 | List | Set | SortedSet |
|---|---|---|---|
| 排序方式 | 按添加顺序 | 无法排序 | 按 score 值排序 |
| 唯一性 | 不唯一 | 唯一 | 唯一 |
| 查找方式 | 按索引/首尾 | 按元素 | 按元素 |

**选择 SortedSet：** 唯一性保证同一用户不重复记录，score 存时间戳实现按时间排序。

## 五、踩坑避坑

### 坑1：isBlogLiked() 未登录空指针

**问题：** `UserHolder.getUser()` 返回 null 时直接调 `.getId()` 会空指针

**后果：** 未登录用户访问 `/blog/hot` 或 `/blog/{id}` 报 500

**修复：** 加判空提前返回
```java
UserDTO user = UserHolder.getUser();
if (Objects.isNull(user)){
    return;
}
```

### 坑2：queryBlogLikes() 多余代码

**问题：** 方法里有 `Long userId = UserHolder.getUser().getId()` 但没用到

**后果：** 未登录用户调排行榜接口空指针

**修复：** 删除多余代码

## 六、效果验证

接口：
```
GET /blog/likes/{id}
```

返回 Top5 点赞用户列表（按点赞时间排序）。

## 七、全文总结

1. SortedSet 比 Set 多了"按 score 排序"能力，适合排行榜场景
2. score 存时间戳，既能去重又能按时间排序
3. 判断是否存在的命令：`SISMEMBER` → `ZSCORE`（判断是否为 null）
4. 注意未登录用户的空指针防护
