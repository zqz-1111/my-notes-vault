# 【踩坑】前后端对接常见问题汇总

## 一、写在前面

前后端分离开发中，接口写好了不等于前端能正常用。这篇记录 hm-dianping 项目中遇到的前后端对接坑，都是实战中真实踩过的。

## 二、接口参数对不上

### 2.1 问题：后端改了方法签名，前端没跟着改

**场景：** 后端 `queryNearby` 方法参数顺序从 `(typeId, x, y, current)` 改成 `(typeId, current, x, y)`，但前端发请求时参数名是对的，顺序无所谓（HTTP 参数是 key=value，不靠顺序）。

**结论：** HTTP 请求参数靠 **参数名** 匹配，不靠顺序。后端 `@RequestParam("typeId")` 取的是 `typeId` 这个 key，跟前端传的顺序无关。

### 2.2 问题：required 参数不匹配

```java
// 后端：x 和 y 是可选的
@RequestParam(value = "x", required = false) Double x
@RequestParam(value = "y", required = false) Double y
```

```javascript
// 前端：始终传了 x 和 y
params: {
    typeId: 0,
    current: 1,
    x: 120.149993,
    y: 30.334229
}
```

**踩坑点：** 如果后端 `required = true`（默认），前端不传就会报 400 Bad Request。设成 `required = false` 更安全，后端做空值判断降级处理。

## 三、axios 响应拦截器的坑

### 3.1 问题：前端收到的数据结构和后端返回的不一样

后端返回：
```json
{"success": true, "data": [{"id": 1, "name": "xxx", "distance": 1.07}, ...]}
```

但前端 `then(({data}) => ...)` 里的 `data` 直接就是数组，不是整个 Result 对象。

**原因：** `common.js` 里有 axios 响应拦截器：

```javascript
axios.interceptors.response.use(function (response) {
    // 判断执行结果
    if (!response.data.success) {
        return Promise.reject(response.data.errorMsg)
    }
    return response.data;  // ← 这里返回了 response.data
}, ...)
```

拦截器把 `response.data`（即 `{success:true, data:[...]}`）返回了，所以 `.then(({data}) => ...)` 解构出来的 `data` 是 `{success:true, data:[...]}` 的 `data` 属性，也就是数组本身。

**数据流向：**
```
后端返回 {success:true, data:[...]}
    ↓ axios 包装
response = {data: {success:true, data:[...]}, status:200, ...}
    ↓ 拦截器 return response.data
{success:true, data:[...]}
    ↓ .then(({data}) => ...)
data = [...]  ← 最终拿到的是数组
```

### 3.2 解决方案

了解拦截器的逻辑后，前端直接用 `data` 就是数组，不需要再 `.data` 取一次。

## 四、距离单位不一致

### 4.1 问题：后端返回 km，前端按 m 显示

后端返回：`"distance": 1.0673`（单位：km）

前端原始代码：
```html
<span v-if="s.distance">
    {{s.distance < 1000 ? s.distance.toFixed(1) + 'm' : (s.distance/1000).toFixed(1) + 'km'}}
</span>
```

这段代码假设 distance 单位是**米**（< 1000 显示 m，否则显示 km）。但后端返回的是 km，1.0673 < 1000，所以显示 "1.1m"，明显不对。

### 4.2 解决方案

```html
<span v-if="s.distance">
    {{s.distance < 1 ? (s.distance * 1000).toFixed(0) + 'm' : s.distance.toFixed(1) + 'km'}}
</span>
```

- `< 1 km` → 乘 1000 转成米，显示 "800m"
- `>= 1 km` → 直接显示 "1.1km"

## 五、排序按钮导致数据混乱

### 5.1 问题：点排序按钮后，数据顺序乱了

前端排序按钮逻辑：
```javascript
sortAndQuery(sortBy) {
    this.params.sortBy = sortBy;
    this.queryShops();  // 直接追加，没清空旧数据
}
```

**问题：** 点"距离"按钮时，`queryShops()` 又请求了第 1 页数据，但没清空 `this.shops`，新数据直接 `concat` 到旧数据后面，导致：
```
旧数据：[A(0.17km), B(1.0km), C(1.0km), D(1.05km), E(1.74km)]
新数据：[A(0.17km), B(1.0km), C(1.0km), D(1.05km), E(1.74km)]
合并后：[A, B, C, D, E, A, B, C, D, E]  ← 重复且顺序错乱
```

### 5.2 解决方案

```javascript
sortAndQuery(sortBy) {
    this.params.sortBy = sortBy;
    this.params.current = 1;    // 重置页码
    this.shops = [];            // 清空旧数据
    this.queryShops();
}
```

## 六、滚动加载不触发

### 6.1 问题：店铺只显示 5 个，往下滚没反应

前端 `@scroll="onScroll"` 绑定在 `.shop-list` 容器上，CSS 设置了 `height: 87%; overflow-y: auto`。

**原因：** 5 个店铺卡片（每个约 120px）总高度约 600px，但容器高度是视口的 87%（约 700px），内容没超出容器，**根本没有滚动条**，scroll 事件不会触发。

### 6.2 解决方案

增大每页加载数量：`DEFAULT_PAGE_SIZE` 从 5 改成 10，内容超出容器后滚动条出现，触底加载就能正常工作。

## 七、浏览器缓存导致改了代码没效果

### 7.1 问题：改了前端 HTML，刷新页面没变化

nginx 返回静态文件时，浏览器会缓存。普通刷新（F5）可能用缓存。

### 7.2 解决方案

- `Ctrl + F5`：强制刷新，跳过缓存
- 或者在 nginx 配置中禁用缓存：
```nginx
location / {
    root html/hmdp;
    add_header Cache-Control "no-cache, no-store";
}
```

## 八、数据库查询打乱排序

### 8.1 问题：Redis 返回的距离顺序是对的，但前端显示乱了

Redis GEORADIUS 返回：`[A(0.17km), B(1.0km), C(1.0km), D(1.05km), E(1.74km)]`

但 `listByIds([A,B,C,D,E])` 查出来可能是：`[A, C, E, B, D]`（数据库按主键排序）

### 8.2 解决方案

```java
String idStr = StrUtil.join(",", ids);  // "1,6,4,8,5"
List<Shop> shops = query()
        .in("id", ids)
        .last("ORDER BY FIELD(id," + idStr + ")")  // 强制按 ID 列表顺序
        .list();
```

`ORDER BY FIELD(id, 1,6,4,8,5)` 让 MySQL 按传入的 ID 顺序返回结果，保持 Redis 的距离排序。

## 九、总结

| 问题 | 根因 | 解决 |
|------|------|------|
| 参数对不上 | required 默认 true | 设 required=false + 后端判空 |
| 数据结构不对 | axios 拦截器返回 response.data | 了解拦截器逻辑 |
| 距离单位错 | 后端 km，前端当 m 处理 | 统一单位，<1 显示 m |
| 排序乱 | 没清空旧数据 | 清空 shops + 重置 current |
| 滚动不触发 | 内容没超出容器 | 增大 pageSize |
| 改了没效果 | 浏览器缓存 | Ctrl+F5 |
| 查询不保序 | 数据库默认排序 | ORDER BY FIELD |
