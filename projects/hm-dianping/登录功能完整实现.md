# 登录功能完整实现：验证码校验 + Token 机制 + 拦截器

## 一、写在前面

1. 为什么学这个 — 登录是任何系统的核心功能，hm-dianping 项目用 Redis + Token 实现无状态登录，比 Session 更适合分布式场景
2. 学完这篇你能掌握：验证码校验、Token 生成与存储、登录拦截器、UserHolder 线程级用户上下文

## 二、前置准备

- 已完成验证码发送功能（见 [[登录功能实现-验证码发送]]）
- Redis 已启动，`StringRedisTemplate` 已注入
- 了解 `RedisConstants` 中的 key 常量

## 三、正文分步讲解

### 3.1 验证码校验（从 Redis 获取）

```java
// 从 Redis 获取验证码
String cacheCode = stringRedisTemplate.opsForValue().get(LOGIN_CODE_KEY + phone);
String code = loginForm.getCode();

// 校验
if (cacheCode == null || !cacheCode.equals(code)) {
    return Result.fail("验证码错误");
}
```

**关键点：** 验证码存 Redis 而不是 Session，好处是：
- 支持分布式部署
- 可以设置 TTL 自动过期（2分钟）

### 3.2 用户查询与自动注册

```java
// 根据手机号查询用户
User user = query().eq("phone", phone).one();

// 不存在则自动注册
if (user == null) {
    user = createUserWithPhone(phone);
}
```

**踩坑记录：** 
```java
// ❌ 错误写法（查询 password 字段）
.eq(User::getPassword, phone)

// ✅ 正确写法（查询 phone 字段）
.eq(User::getPhone, phone)
```

这个 Bug 会导致每次登录都注册新用户！

### 3.3 生成 Token 并存入 Redis

```java
// 1. 生成随机 Token
String token = UUID.randomUUID().toString(true);

// 2. User 转 UserDTO（去掉密码等敏感字段）
UserDTO userDTO = BeanUtil.copyProperties(user, UserDTO.class);

// 3. UserDTO 转 HashMap（Redis Hash 存储需要）
Map<String, Object> userMap = BeanUtil.beanToMap(userDTO, new HashMap<>(),
        CopyOptions.create()
                .setIgnoreNullValue(true)
                .setFieldValueEditor((fieldName, fieldValue) -> fieldValue.toString()));

// 4. 存入 Redis
String tokenKey = LOGIN_USER_KEY + token;
stringRedisTemplate.opsForHash().putAll(tokenKey, userMap);

// 5. 设置过期时间
stringRedisTemplate.expire(tokenKey, LOGIN_USER_TTL, TimeUnit.MINUTES);

// 6. 返回 Token 给前端
return Result.ok(token);
```

**关键点：**
- Token 用 `UUID.randomUUID().toString(true)` 生成，去掉横杠
- UserDTO 比 User 安全，不包含密码
- Redis Hash 结构存储用户信息，方便后续读取

### 3.4 LoginInterceptor 拦截器

```java
public class LoginInterceptor implements HandlerInterceptor {

    private StringRedisTemplate stringRedisTemplate;

    public LoginInterceptor(StringRedisTemplate stringRedisTemplate) {
        this.stringRedisTemplate = stringRedisTemplate;
    }

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        // 1. 获取请求头中的 token
        String token = request.getHeader("authorization");
        if (StrUtil.isBlank(token)) {
            response.setStatus(401);
            return false;
        }

        // 2. 从 Redis 获取用户信息
        String key = LOGIN_USER_KEY + token;
        Map<Object, Object> userMap = stringRedisTemplate.opsForHash().entries(key);
        if (userMap.isEmpty()) {
            response.setStatus(401);
            return false;
        }

        // 3. 转换为 UserDTO
        UserDTO userDTO = BeanUtil.fillBeanWithMap(userMap, new UserDTO(), false);

        // 4. 存入 UserHolder（ThreadLocal）
        UserHolder.saveUser(userDTO);

        // 5. 刷新 Token 有效期
        stringRedisTemplate.expire(key, LOGIN_USER_TTL, TimeUnit.MINUTES);

        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        // 移除用户，避免内存泄漏
        UserHolder.removeUser();
    }
}
```

**关键点：**
- 构造函数注入 `StringRedisTemplate`（不能用 @Resource，因为是 new 出来的）
- 每次请求刷新 Token 有效期（滑动过期）
- `afterCompletion` 中移除 ThreadLocal，防止内存泄漏

### 3.5 MvcConfig 配置拦截器

```java
@Configuration
public class MvcConfig implements WebMvcConfigurer {

    @Resource
    private StringRedisTemplate stringRedisTemplate;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new LoginInterceptor(stringRedisTemplate))
                .excludePathPatterns(
                        "/shop/**",
                        "/voucher/**",
                        "/shop-type/**",
                        "/upload/**",
                        "/blog/hot",
                        "/user/code",
                        "/user/login"
                ).order(1);
    }
}
```

**关键点：** 排除不需要登录的路径（验证码、登录、公开接口）

### 3.6 UserHolder 工具类

```java
public class UserHolder {
    private static final ThreadLocal<UserDTO> tl = new ThreadLocal<>();

    public static void saveUser(UserDTO user) {
        tl.set(user);
    }

    public static UserDTO getUser() {
        return tl.get();
    }

    public static void removeUser() {
        tl.remove();
    }
}
```

**用途：** 在同一个请求内任何位置都能获取当前登录用户，不用每次都查 Redis

## 四、踩坑避坑指南

**坑 1：User 类型转换错误**
- 报错信息：`'saveUser(UserDTO)' 无法应用于 '(User)'`
- 原因分析：`UserHolder.saveUser()` 需要 `UserDTO`，但传了 `User`
- 解决方案：用 `BeanUtil.copyProperties(user, UserDTO.class)` 转换

**坑 2：查询条件写错**
- 现象：每次登录都注册新用户
- 原因：`.eq(User::getPassword, phone)` 查的是密码字段
- 正确写法：`.eq(User::getPhone, phone)`

**坑 3：Session 和 Redis 混用**
- 现象：登录成功但拦截器返回 401
- 原因：登录用 Redis 存用户，拦截器从 Session 取用户
- 解决：统一用 Redis + Token 机制

## 五、效果验证

1. 发送验证码：`POST /user/code`，Redis 中出现 `login:code:手机号`
2. 登录：`POST /user/login`，返回 token，Redis 中出现 `login:token:xxx`
3. 访问 `/user/me`：请求头带 `authorization: token`，返回用户信息
4. 不带 token 访问：返回 401

## 六、全文总结

- 验证码存 Redis 用 `String` 结构，设置 2 分钟 TTL
- 用户信息存 Redis 用 `Hash` 结构，Token 作为 key
- LoginInterceptor 从请求头取 Token，从 Redis 取用户，存入 UserHolder
- UserHolder 用 ThreadLocal 实现，同一线程内共享用户信息
- 记得在 `afterCompletion` 中清理 ThreadLocal
