# 选题讨论

## 一、方向梳理

### 已排除的方向

| 方向 | 排除原因 |
|---|---|
| 纯知识图谱构建（无改进） | 创新性不足，EI 难发 |
| 纯算法（链接预测/嵌入） | 大论文不好衔接，偏理论 |
| 综述类 | 创新性要求高，容易写空 |

### 候选方向

#### 方向 A：领域知识图谱构建 + 关系抽取改进

- **做法**：选一个具体领域，构建该领域的知识图谱，在关系抽取环节用某个改进策略（加注意力机制、改进预训练模型微调方式等）
- **优点**：工作量可控，创新点明确（"改进了XX方法在XX领域的抽取效果"），大论文可以直接扩展为"面向XX领域的知识图谱构建与应用研究"
- **大论文衔接**：可以加问答、推理、可视化等模块撑大论文
- **可行性**：★★★★★

#### 方向 B：知识图谱 + 大模型增强问答（RAG）

- **做法**：构建小型领域知识图谱，接入大模型做 RAG 增强问答，技术点在检索策略或图谱融合方式上做改进
- **优点**：和大模型结合是当前热点，审稿人印象好
- **风险**：需要有大模型调用的实验环境
- **大论文衔接**：可以扩展为知识图谱驱动的智能问答系统
- **可行性**：★★★★

#### 方向 C：知识图谱补全/链接预测

- **做法**：在公开数据集上做链接预测，改进 embedding 模型或加入多模态信息
- **优点**：数据集现成，不需要自己从零建图谱
- **缺点**：纯算法方向，大论文衔接弱
- **可行性**：★★★

## 二、推荐结论

**暂定方向 A**：领域知识图谱构建 + 关系抽取改进

理由：
1. 自己构建图谱 = 有完整工作量，答辩好讲
2. 改进关系抽取 = 有技术创新点，够发 EI
3. 大论文直接扩展 = 加推理、问答、应用模块就能撑起来
4. 中文领域数据集不难找，不需要大规模算力
