# 论文17：KGLM — 微调LLM构建肺癌知识图谱

## 基本信息

- **标题**：Fine-tuned Large Language Models with Structured Prompts Enable Efficient Construction of Lung Cancer Knowledge Graphs
- **作者**：Zhou Chunfang, Gong Qingyue, Luan Huidan, Zhan Wendong, Zhu Jinyang, Qi
- **发表**：Nature Scientific Reports, 2026
- **链接**：https://www.nature.com/articles/s41598-026-38959-w

## 一、写在前面

### 解决什么问题

传统KG构建需要大量人工标注，成本高。论文用微调LLM + 结构化提示词自动抽取三元组，F1=82%，比baseline提升25%。

### 核心思路

端到端生成式抽取：原始文本 → 结构化提示词 → QLoRA微调的ChatGLM-6B → 三元组

## 二、整体架构

### 三阶段框架

```
非结构化文本 → [信息抽取] → 三元组
                              ↓
                         [知识融合（实体对齐）]
                              ↓
                         [Neo4j存储]
```

### 与传统Pipeline的区别

| 对比项 | 传统NER+RE Pipeline | KGLM端到端 |
|---|---|---|
| 流程 | NER → RE 两步 | 一步生成三元组 |
| 错误传播 | NER错则RE全错 | 联合抽取，无级联错误 |
| 灵活性 | 受限于分类器 | 生成式，处理复杂句式 |
| 数据需求 | 大量标注数据 | Few-Shot + 微调 |

## 三、核心方法

### 3.1 微调策略：QLoRA

- 基座模型：ChatGLM-6B
- 量化：8-bit量化，显存从24GB降至6GB
- LoRA：侧支引入低秩矩阵，只训练适配器
- 硬件：单张RTX 4090即可

### 3.2 结构化提示词设计（核心创新）

提示词由4个组件构成：

**a. 系统角色设定**
- 要求模型扮演"KG领域专业NLP专家"

**b. 三元组定义+Few-Shot**
- 给出三元组的具体定义和示例

**c. 输出格式约束**
- 严格限定JSON格式
- 相同头实体的三元组组合在一起分行输出
- 不加约束时F1仅61%（自由生成）

**d. 输出规则+思维链（CoT）**
- 以疾病为头实体，属性为尾实体
- 引入"Let's think step by step"处理嵌套

### 3.3 实体对齐：Jaccard + SBERT

**二阶段混合算法：**

1. **粗筛（Jaccard）**：字面重合度计算，速度快，低于阈值t1直接判定不同
2. **精算（SBERT）**：语义向量+余弦相似度，解决"慢阻肺"="COPD"等语义同义词

**对中药实体的启发**：黄芪vs炙黄芪需要特殊处理——可作为"炮制变体"通过关系边连接，而非盲目合并。

### 3.4 存储：Neo4j + APOC

规范化三元组写入Neo4j图数据库。

## 四、实验结果

### 数据集

- 总规模：49,860条Prompt-Response对
- 非结构化网页数据（59.9%）：好大夫在线、寻医问药等5,000条
- 半结构化临床数据（21.1%）：周仲瑛临床医案
- 结构化公共图谱数据（19.0%）：开源医疗KG

### 主要结果

| 模型 | RE F1 |
|---|---|
| 原始ChatGLM-6B（无微调） | 57% |
| CNN | 65% |
| CNN + Attention | 69% |
| BERT | 73% |
| BERT + Attention | 77% |
| **KGLM + Prompt** | **82%** |

关键：F1=82%是关系抽取任务，相对baseline提升25%。

## 五、对我们项目的启发

### 可以直接借鉴的

1. **LLM辅助生成训练数据**
   - 用少量种子数据（200条）+LLM批量生成候选
   - 人工校验过滤，标注效率提升数倍
   - 适用阶段：第5-6周数据处理

2. **CoT提示词处理嵌套**
   - "一步步思考"：症状→病机→治法→方剂→加减
   - 可迁移到中医医案的三元组抽取

3. **实体对齐方法**
   - Jaccard+SBERT两阶段
   - 处理TCMSP数据中的同义词问题

4. **输出格式约束**
   - 不加约束F1仅61%，加JSON约束后大幅提升
   - 数据标注流程中必须严格定义格式

### 两条技术路线的对比

| 路线 | 优势 | 劣势 |
|---|---|---|
| BERT-CRF做NER+RE | 边界精准、推理快、成本低 | 无法处理重叠实体、错误传播 |
| LLM端到端抽取 | 语义理解强、联合抽取、泛化好 | 需GPU、可能幻觉 |

**可能的混合方案**：BERT-CRF做NER（精准边界）+ LLM辅助RE（语义理解优势）

### 提示词设计模板（可复用）

```
角色：具备20年临床经验的中医内科学专家及知识图谱专家
Schema：[疾病,主治,方剂]、[方剂,组成,中药]、[中药,炮制方法,变体]
CoT：症状→诊断病机→治法→方剂→随症加减药味
输出：严格JSON格式，每行一个三元组
```

## 六、全文总结

1. KGLM用QLoRA微调ChatGLM-6B，端到端抽取三元组，F1=82%
2. 结构化提示词是核心：角色+定义+格式约束+CoT
3. 实体对齐用Jaccard+SBERT两阶段，处理同义词
4. 不加格式约束F1仅61%，约束极其重要
5. LLM辅助生成训练数据是解决标注数据匮乏的最佳路径
