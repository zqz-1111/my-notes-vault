# 论文12：DG-SpanTCM — 基于双粒度特征增强与片段分类的中医EMR嵌套NER

## 基本信息

- **标题**：Nested Named Entity Recognition in Traditional Chinese Medicine Electronic Medical Records via Dual-Granularity Feature Augmentation and Span Classification
- **作者**：Minghao, Luo, Qing, Ye, Chunlei, Cheng, Peng, Lin
- **发表**：Nature Scientific Reports, 2025
- **链接**：https://www.nature.com/articles/s41598-025-22283-w

## 一、写在前面

### 解决什么问题

中医电子病历中存在大量嵌套实体（如"加减归脾汤"包含"归脾汤"），传统BIO序列标注无法处理。论文提出DG-SpanTCM框架，用Span分类替代序列标注来解决嵌套问题。

### 核心贡献

1. 字词双粒度特征融合（BERT + SoftLexicon）
2. 对抗训练增强鲁棒性（FGM）
3. Span二维矩阵解码处理嵌套
4. ACW Loss解决长尾分布

## 二、为什么做？（背景与痛点）

### 中医EMR的三大挑战

**挑战1：领域术语复杂**
- 混杂古汉语（"脉弦细"）、西医术语（"CRP升高"）、复合处方（"柴胡疏肝散合六味地黄丸"）
- 基层医院常用简称，语义鸿沟大
- 通用预训练模型在中医任务上F1下降近20%

**挑战2：嵌套实体（本文核心）**
- 经典中医文献中63%的实体存在嵌套
- 现代EMR中嵌套率高达72%（中西医术语共存）
- 例子："加减归脾汤治疗心脾两虚型失眠" → "加减归脾汤"嵌套"归脾汤"
- BIO标注同一位置只能给一个标签，无法表示嵌套

**挑战3：长尾分布与噪声**
- 58%的罕见证候一年出现<5次
- 高频实体占70%数据量
- 不同医生书写风格差异大，存在错别字和标注噪声

## 三、怎么解决的？（模型架构）

### 整体数据流

```
输入文本
  ├─► BERT编码 → 字符向量 X^c ──┐
  │                              ▼
  └─► SoftLexicon(BMES) → TF-IDF权重 → 词向量 X^w
                                          │
       双粒度融合(Concat+LayerNorm) ◄────┘
              │
              ▼
       对抗训练(FGM扰动)
              │
              ▼
       BiLSTM(双向语义融合)
              │
       ├─► FFNN_Start ──┐
       │                ▼
       └─► FFNN_End ───► 乘法注意力机制
                            │
                            ▼
                     MLP片段分类
                            │
                            ▼
                    ACW Loss + ARGMAX
                            │
                            ▼
                      输出实体结果
```

### 模块1：双粒度编码层

**字级别**：BERT给每个字生成上下文向量

**词级别**：SoftLexicon + 中医词典
- 为每个字构建BMES四元组：
  - B（Begin）：以该字开头的词集合
  - M（Middle）：包含该字的中间词集合
  - E（End）：以该字结尾的词集合
  - S（Single）：该字单独成词
- 例子："肠"在"右半结肠癌"中 → B={"肠癌"}, M={"结肠癌"}, E={"结肠","半结肠","右半结肠"}, S={"肠"}

**TF-IDF权重衰减**：高频无用词权重降低，罕见专业词权重放大

**为什么不用jieba**：硬切分会破坏嵌套结构，错误传播。SoftLexicon是软匹配，保留所有潜在词汇可能性。

### 模块2：对抗训练（FGM）

- 在Embedding层添加微小扰动Δx
- 模拟worst-case，增强对错别字、缩写的鲁棒性
- 用BiLSTM融合后提取双向长距离上下文

### 模块3：Span分类解码

**核心思路**：抛弃CRF，改用Span分类

- n个字的句子，枚举所有可能的span（起止位置对）
- span总数 = n(n+1)/2
- 解码矩阵W是n×n的上三角矩阵
- 每个元素W_ij存储span(i,j)的预测类别
- 引入乘法注意力机制捕捉span内部依赖
- MLP对每个span分类：是哪类实体，还是非实体

**解码过程**：遍历上三角 → 非"O"的坐标(i,j) → 截取文本[i:j] → 输出实体

### 模块4：ACW Loss（自适应类别权重损失）

**解决的问题**：长尾分布导致模型只认常见病，无视罕见病

**机制**：
- 根据实体类别频率动态计算惩罚权重
- 越罕见的类别，猜错惩罚越大
- γ参数控制调整剧烈程度
- 配合噪声感知训练策略

## 四、效果如何？（实验结果）

### 数据集

| 数据集 | 来源 | 规模 | 实体数 | 嵌套率 | 句长 |
|---|---|---|---|---|---|
| 自建TCM | 江西中医药大学 2014-2021 | 42,500句 | 43,028 | 52.3% | 5.8字 |
| CMeEE | 公开医疗数据集 | 6,500条 | - | 31.8% | 13字 |

标注：5人双盲标注，BRAT工具，Kappa>0.85，17类实体

### 主要结果

| 模型 | TCM F1 | CMeEE F1 |
|---|---|---|
| BERT-CRF | 较低 | 较低 |
| BERT-Biaffine | 较高 | 较高 |
| Global Pointer | 次高 | 次高 |
| **DG-SpanTCM** | **70.53%** | **70.51%** |

### 消融实验

| 变体 | 去掉模块 | F1变化 |
|---|---|---|
| our1 | 对抗训练 | **-3.96%**（影响最大） |
| our2 | 词典匹配 | -2.71% |
| our3 | 两者都去 | 进一步下降 |
| our4 | ACW→交叉熵 | -2.63%（精准率大幅下滑） |

### 8个Baseline

1. BERT-CRF：序列标注经典
2. Lattice-LSTM：晶格结构融合词汇
3. SoftLexicon-LSTM：软融入词汇信息
4. PFN：分区过滤网络
5. BERT-Biaffine：双仿射解码器
6. Global Pointer：旋转位置编码+全局指针
7. AT-GBGP：Global Pointer + 对抗训练
8. BERT-BiLSTM-CRF：经典加强版

## 五、对我们项目的启示

### 可以借鉴的

1. **SoftLexicon的软融合范式**：药理特征向量可以用同样的Concatenate+LayerNorm方式注入BERT
2. **TF-IDF动态权重**：防止"甘草"等万金油中药的药理特征泛滥
3. **Span分类架构**：如果嵌套问题严重，可考虑替换CRF
4. **对抗训练FGM**：小数据集提升明显，实现简单

### 创新点的三层递进

| 层次 | 内容 | 来源 |
|---|---|---|
| 第1层：字词边界 | SoftLexicon词典匹配 | 论文12（已做到） |
| 第2层：药理特征 | OB/DL等药理属性融合 | 我们的原创方向 |
| 第3层：KG结构先验 | 配伍关系等图谱拓扑 | 论文15启发 |

论文12只做到了第1层，我们做到第2层就有差异化，三层全做到是完整故事。

### 关键差异

- 他们的"双粒度"是**结构与边界维度**（告诉模型词从哪开始到哪结束）
- 我们的"药理特征融合"是**内在属性维度**（让模型理解药名背后的药理逻辑）
- 方向相同但深度不同，我们的创新点更深一层

## 六、全文总结

1. DG-SpanTCM用Span分类替代BIO标注，天然支持嵌套实体识别
2. BERT+SoftLexicon实现字词双粒度编码，解决中医术语边界问题
3. FGM对抗训练对小数据集提升最大（-3.96%），是最关键模块
4. ACW Loss通过动态权重解决长尾分布，罕见实体识别提升显著
5. F1=70.53%（自建TCM数据集），显著超越所有baseline
