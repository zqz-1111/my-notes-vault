## 七、2025-2026 关键趋势

1. **LLM + KG 是当前主流** — 很多人在用大模型做实体/关系抽取
2. **知识蒸馏** — 用大模型指导小模型，轻量化部署（Nature 2026）
3. **混合架构** — ML 做实体识别 + LLM 做关系抽取（He et al. 2025）
4. **多模态融合** — 文本 + 药理学特征 + 古籍知识（未来方向）

## 八、论文1 综述核心收获（2026-07-18 阅读）

### 技术演进路线
```
基于规则/词典 → CRF → BiLSTM-CRF → BERT-BiLSTM-CRF → 我们的改进方法
```

### 综述提到的挑战 = 我们的创新切入点

| 综述挑战 | 我们的方案 |
|---|---|
| 缺乏大规模标准化标注语料 | 用 TCMNER2026（22万实体）+ TCMSP 结构化数据 |
| 嵌套实体难处理 | 药理学特征辅助消歧 |
| 多模态信息未融合 | **核心创新：把药理学数值特征（OB/DL）作为"数值模态"融入文本模型** |

### 创新点表述（基于综述支撑）
> 综述 [1] 指出，将中药药理学特征作为独立模态融入知识图谱构建是未来重要方向。本文在此基础上，提出一种融合药理学特征（OB、DL）的中医文本关系抽取方法，在 BERT 预训练模型基础上，通过注意力机制动态融合文本语义特征与中药药理学特征，有效提升了中医领域关系抽取的准确率。

## 九、论文15 阅读收获（2026-07-18）

### 论文概要
- **标题**：Integrating KG with Ancient Chinese Medicine Classics
- **来源**：Chinese Medicine（Springer），2025年10月，医学一区/中医学顶刊
- **核心方案**：MAS（多智能体系统）+ RAG（检索增强生成）+ KG 协同架构
- **技术路线**：顶层本体设计 → 多Agent流水线（翻译/对齐/NER/RE/校验）→ Neo4j存储

### 古籍文本五大难点（对我们选题有指导意义）
1. 缺乏统一术语系统
2. 文法结构灵活多变（倒装、省略）
3. 修辞手法复杂（互文、隐喻）
4. 同义词/文本相似混淆严重
5. 复杂嵌套实体结构

### 现有技术baseline
- 词向量层：Word2Vec / BERT / ALBERT
- NER层：BERT-BiLSTM-CRF（公认最优baseline）
- 关系分类层：朴素贝叶斯 / 规则模板匹配
- 图存储：Neo4j

### 我们和他们的区别
| 维度 | MAS方案 | 我们的BERT改进 |
|---|---|---|
| 层次 | 宏观系统工程 | 底层算法改进 |
| 古文处理 | 先翻译再处理 | 直接在原文上增强 |
| 模型规模 | 大模型（贵、慢、有幻觉） | 轻量BERT（快、确定性高） |

### 三个创新切入点（来自论文局限）
1. **翻译依赖症** → 开发原生面向古籍的改进BERT
2. **大模型幻觉 vs 低容错率** → BERT作为领域小模型过滤/校准大模型输出
3. **嵌套实体识别乏力** → 融合KG先验知识或对比学习

### 创新点方向调整
原方案：融合药理学特征的BERT RE
新方案：**融合药理学特征 + KG结构先验的BERT，提升中医嵌套实体识别和关系抽取准确率**
