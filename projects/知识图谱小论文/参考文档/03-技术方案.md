# 技术方案

## 一、整体架构

```
数据来源：TCMSP + 中医文献 + TCMNER2026
        ↓
数据预处理：清洗、结构化、标注
        ↓
命名实体识别（NER）：BERT-CRF 识别中药/方剂/疾病/症状/成分
        ↓
关系抽取（RE）：融合药理学特征 + KG先验的BERT
        ↓
知识图谱构建：三元组 → Neo4j 存储
        ↓
可视化展示 + 查询验证
```

## 二、实体类型

| 实体类型 | 说明 | 示例 |
|---|---|---|
| Herb（中药） | 单味中药 | 黄芪、当归、人参 |
| Formula（方剂） | 中药处方 | 四物汤、六味地黄丸 |
| Component（成分） | 中药化学成分 | 槲皮素、黄芩苷 |
| Disease（疾病） | 疾病名称 | 冠心病、糖尿病 |
| Symptom（症状） | 临床症状 | 气短、乏力、失眠 |
| Target（靶点） | 蛋白靶点 | PTGS2、ESR1 |

## 三、关系类型

| 关系类型 | 头实体 | 尾实体 | 示例 |
|---|---|---|---|
| contains | Formula | Herb | 四物汤 → contains → 当归 |
| has_component | Herb | Component | 黄芪 → has_component → 槲皮素 |
| treats | Herb/Formula | Disease | 黄芪 → treats → 气虚 |
| targets | Component | Target | 槲皮素 → targets → PTGS2 |
| indicates | Disease | Symptom | 气虚 → indicates → 乏力 |

## 四、创新点

### 当前方案（2026-07-18 更新）

**融合药理学特征 + KG结构先验的BERT，提升中医嵌套实体识别和关系抽取准确率**

三个差异化切入点（来自论文15的局限）：
1. 原生面向古籍文本，不依赖翻译
2. 轻量级BERT，确定性高，可作为大模型的过滤器
3. 融合KG先验知识解决嵌套实体问题

### 模型结构

```
输入：中医文本 + 实体对
        ↓
BERT 编码器 → 文本语义向量 (768维)
        ↓                    ↓
药理学特征嵌入层 ← OB/DL/分子量等数值特征
        ↓
特征融合层（注意力机制）
        ↓
关系分类器 → 关系类型
```

### 对比实验

| 方法 | 说明 |
|---|---|
| BERT-base | 基线 |

### 消融实验

- 去掉注意力机制
- 去掉KG先验知识
