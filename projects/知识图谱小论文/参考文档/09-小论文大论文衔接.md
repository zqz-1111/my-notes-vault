# 小论文→大论文衔接规划

> EI小论文的内容如何扩展为硕士毕业论文的章节。

---

## 一、核心逻辑

小论文是大论文的一个**子集**，大论文需要在此基础上做**系统性扩展**。

- **小论文**：4-6页，聚焦一个核心贡献，实验够用即可
- **大论文**：60-80页，需要完整的问题定义、方法、实验、系统展示

---

## 二、章节映射

| 小论文内容 | 大论文章节 | 扩展要点 |
|---|---|---|
| Introduction | 第1章 绪论 | 加研究背景（中医药信息化现状）、研究意义、论文组织结构 |
| Related Work | 第2章 文献综述 | 扩展到20-30页，分类更细（NER综述、RE综述、KG构建综述、领域应用综述） |
| NER方法（创新一） | 第3章 基于词汇知识增强的中医NER | 展开SoftLexicon细节、更多消融实验、参数敏感性分析、错误分析 |
| RE方法（创新二+三） | 第4章 融合药理特征与KG先验的中医RE | 展开融合机制设计、注意力可视化、更多baseline对比 |
| LLM辅助数据构建 | 第5章 基于LLM的中医RE数据自动构建 | 独立成章，展开Prompt设计、Self-consistency分析、人工评估 |
| 实验 + Case Study | 第6章 实验与结果分析 | 补充更多实验（跨数据集、不同划分、统计显著性检验） |
| — | 第7章 中医知识图谱系统设计与实现 | **大论文新增**：系统架构、接口设计、Neo4j存储、可视化Demo |
| — | 第8章 总结与展望 | **大论文新增**：工作总结、创新点总结、未来方向 |

## 三、小论文已覆盖 vs 大论文需补充

### 已覆盖（可直接复用）

- [x] KG先验方案（TransE/RotatE图嵌入）

### 大论文需额外补充

| 补充项 | 说明 | 优先级 |
|---|---|---|
| 文献综述扩展 | 从3-4页扩展到20-30页，覆盖NER/RE/KG各方向的代表性工作 | 高 |
| 系统设计章节 | Neo4j存储设计、API接口、可视化前端、部署方案 | 高 |
| 更多消融实验 | 小论文受限于篇幅，大论文每层创新需要更完整的消融 | 高 |
| 参数敏感性分析 | 学习率、batch size、图嵌入维度、药理特征维度等 | 中 |
| 统计显著性检验 | 配对t-test或McNemar检验，证明改进不是随机波动 | 中 |
| 跨数据集验证 | 在CMeEE等其他数据集上验证方法的泛化性 | 中 |
| 应用场景Demo | 基于KG的问答系统或处方推荐系统原型 | 低 |
| 错误分析深入 | 每类错误的详细分析+改进方向 | 中 |

## 四、时间规划建议

| 阶段 | 小论文进度 | 大论文同步工作 |
|---|---|---|
| 第1-4周 | 文献调研 | 综述初稿同步写 |
| 第5-7周 | 数据收集 | 数据部分可直接复用 |
| 第8-12周 | 模型实验 | 实验结果直接进大论文第3-6章 |
| 第13-15周 | 小论文写作 | 同步写大论文方法章节 |
| 第16周 | 投稿 | 大论文框架基本成型 |
| 投稿后 | 等结果 | 补充系统Demo + 更多实验 + 润色 |

**关键原则**：小论文实验结果直接进大论文，不需要重复做。大论文额外补的是**深度和广度**，不是重新做一遍。
