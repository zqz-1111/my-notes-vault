## 一、研究问题（Research Gap）

### 1.1 现有工作的不足

| 问题 | 具体表现 | 现有工作如何处理 | 遗留问题 |
|---|---|---|---|
| **NER词边界模糊** | 中医临床文本缺乏自然分词，"补气固表"可拆为"补气"+"固表"或整体 | BiLSTM-CRF仅依赖字符序列，无外部词汇知识 | 长实体和复合实体识别差 |
| **RE缺乏领域知识** | 中药关系（如配伍、归经）依赖药理学特性，纯文本模型无法捕获 | BERT/RE模型仅用文本语义，忽略药理学参数 | 对"为什么有这个关系"缺乏解释 |
| **RE缺乏结构先验** | 已有知识图谱包含大量先验关系，但未被用于指导RE | 现有中医RE方法不利用已有KG结构 | 浪费了TCM_KG等资源的先验知识 |
| **RE数据稀缺** | 中医领域没有公开的RE标注数据集 | 依赖人工标注，成本高、规模小 | 无法训练大规模RE模型 |

### 1.2 我们的切入点

**核心观察**：中医知识图谱构建是一个**管道任务**（pipeline），NER和RE是两个子任务，但现有工作将它们割裂处理——NER只看文本不看知识，RE只看文本不看药理和结构。

**我们的思路**：在管道的每个阶段注入**领域特异性知识**：
- NER阶段 → 注入**词汇知识**（词典增强）
- RE数据构建 → 注入**LLM知识**（大模型辅助标注）
- RE模型 → 注入**药理知识 + 结构知识**（TCMSP + KG先验）

## 二、三个创新点

### 创新一：基于SoftLexicon词典增强的中医NER

**一句话**：将中医专业词典通过SoftLexicon机制融入BERT编码，增强中医实体的边界识别能力。

**技术要点**：
- 对每个字符位置，用BMES四元组编码所有匹配到的词典词（而非jieba硬切分）
- 词向量拼接到BERT字向量上，再送入CRF解码
- 词典来源：TCM_KG实体词表(~17,000词) + 中医专业词典

**为什么是创新**：
- SoftLexicon本身（2020 ACL）不是新方法，但在**中医领域**首次应用
- 中医文本的特殊性：古文表述、长实体（如"补气固表活血化瘀"）、专业术语密集，词汇知识的价值比通用领域更大
- 相比jieba分词后拼接的方案，SoftLexicon避免了硬切分的错误传播

**预期贡献**：
- 对TCMNER2026 baseline（BiLSTM-CRF、BERT-CRF）提升2-4% F1
- 对长实体（≥4字）和稀有实体类型（如TCMTM治法）提升更显著

**消融实验**：w/o SoftLexicon → 验证词汇知识的独立贡献

---

### 创新二：融合药理学特征的中医关系抽取

**一句话**：将中药的药理学参数（OB/DL等）编码为特征向量，通过注意力机制与文本语义融合，增强RE模型对"中药-成分""中药-疾病"等关系的识别。

**技术要点**：
- 从TCMSP获取实体的6维药理学特征（OB/DL/MW/AlogP/Hdon/Hacc）
- 归一化后通过全连接层编码为64维特征向量 v_pharm
- 通过注意力机制将药理特征与BERT文本表示融合

**为什么是创新**：
- 现有中医RE方法**全部基于文本语义**，从未引入药理学参数
- 药理学参数提供了"为什么这个成分属于这味药"的科学依据（如高OB=好吸收，高DL=好成药）
- 这是一种**跨模态融合**：文本语义（BERT） + 数值特征（药理参数）

**预期贡献**：
- 对"has_component"（中药-成分）和"treats"（中药-疾病）两类关系提升最显著
- 提供可解释性：注意力权重可展示哪些药理特征对关系分类贡献大

**消融实验**：w/o 药理特征 → 验证药理知识的独立贡献

---

### 创新三：融合知识图谱先验的关系抽取

**一句话**：在TCM_KG上预训练图嵌入（TransE/RotatE/ComplEx），将实体的结构化表示通过注意力机制融入RE模型，利用已有知识图谱的拓扑结构指导关系分类。

**技术要点**：
- 在TCM_KG(~17,000实体)上预训练图嵌入，每个实体得到128维向量
- 对RE输入的实体对，查找其图嵌入拼接为 v_kg
- 通过注意力机制将KG结构信息与文本语义融合

**为什么是创新**：
- 现有中医RE方法**不利用已有KG的结构信息**
- KG先验的价值：如果TCM_KG中已知"黄芪 treats 气虚"，那么在新文本中识别到黄芪和气虚时，KG先验可以提供强信号
- 解决冷启动问题：对于文本中出现较少的实体，KG先验仍可提供可靠的关系判断

**预期贡献**：
- 对低频实体对和稀有关系类型提升最显著
- 跨句子实体对的关系识别能力增强（KG先验弥补文本上下文不足）

**消融实验**：w/o KG先验 → 验证结构知识的独立贡献

## 三、附加创新：LLM辅助RE数据构建

**一句话**：利用大语言模型（GPT-4/ChatGLM）自动标注中医关系数据，通过Self-consistency策略保证标注质量，解决中医RE数据稀缺问题。

**技术要点**：
- 设计结构化Prompt，让LLM判断实体对的关系类型
- Self-consistency策略：Temperature=0、3次采样、Majority Voting
- 人工抽检10-20%样本作为质量校验

**为什么有价值**：
- **方法论贡献**：提出了一种面向专业领域的LLM辅助数据构建范式
- **实用性**：解决了中医RE数据集不存在的问题，使整个pipeline可行
- **可迁移**：同样的方法可迁移到其他专业领域（法律、生物、金融等）

**论文定位**：作为方法论创新单独一节讨论，而非并列为第四个创新点（避免创新点过多引起审稿人质疑）。

## 四、创新点之间的逻辑关系

```
                ┌─────────────────────────────────────┐
                │         中医知识图谱自动构建           │
                │       （Pipeline视角，整体创新）        │
                └──────────────┬──────────────────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         ▼                     ▼                     ▼
    ┌─────────┐         ┌──────────┐          ┌──────────┐
    │ 创新一   │         │ LLM辅助   │          │ 创新二+三 │
    │ NER增强  │───────▶│ RE数据构建 │─────────▶│ RE增强    │
    │ 词汇知识  │         │ 方法论     │          │ 药理+KG   │
    └─────────┘         └──────────┘          └──────────┘
```

**叙事逻辑**：
1. **NER阶段**：先用词汇知识增强把实体识别做好（创新一）
2. **数据构建**：识别出的实体作为LLM标注的输入，自动构建RE训练数据（附加创新）
3. **RE阶段**：用药理知识和KG先验增强关系抽取（创新二+三）
4. **KG输出**：NER+RE的结果组装成知识图谱

这种管道式创新的优势：
- 每个阶段的创新独立可验证（单独消融）
- 整体pipeline形成闭环（文本→实体→关系→图谱）
- 避免审稿人质疑"为什么不端到端"（管道设计有其合理性：阶段解耦便于调试和替换）

## 五、对比现有工作

| 对比维度 | 现有工作 | 我们的方法 |
|---|---|---|
| NER | BiLSTM-CRF / BERT-CRF（纯字符序列） | BERT + SoftLexicon（词汇知识增强） |
| RE数据 | 人工标注或无数据 | LLM自动标注 + Self-consistency |
| RE模型 | 纯BERT文本分类 | BERT + 药理特征 + KG先验 |
| 领域知识利用 | 无 | 三重知识融合（词汇+药理+结构） |
| 可解释性 | 黑盒 | 注意力权重可视化 + 药理特征贡献分析 |

## 八、审稿人可能的质疑与回应

| 质疑 | 回应策略 |
|---|---|
| "创新一SoftLexicon不是新方法" | 强调**领域适配**的贡献：中医文本的特殊性（无分词、长实体、古文）使词汇知识的价值比通用领域更大；首次在中医NER中应用 |
| "创新二+三是简单的特征拼接" | 强调**注意力融合机制**的设计：不是简单concat，而是让模型学习哪些药理特征和KG信息对当前实体对最有用 |
| "为什么不端到端" | 强调**管道设计的合理性**：阶段解耦便于调试、消融、替换；NER和RE的数据来源不同（TCMNER2026 vs LLM标注），端到端反而不好训 |
| "LLM标注质量不可靠" | 引用Self-consistency策略的理论依据（Wang et al., 2023）；报告人工抽检的准确率数据 |
| "三个创新太多，每个都不深" | 三个创新分别对应pipeline的三个阶段，各有独立的消融验证，不是同一个方向的三个变体 |
