# 数据集调研

第3周任务：调研TCMNER2026、ShenNong_TCM_Dataset等数据集可用性。

---

## 一、TCMNER2026（IEEE DataPort）

### 基本信息

| 项目 | 内容 |
|---|---|
| 来源 | 汕头大学 + 广州中医药大学 |
| 发布时间 | 2026-06-27 |
| DOI | 10.21227/s6sh-nc79 |
| 规模 | 221,318 标注实体 / 6,819 临床病历 / 400+位老中医 |
| 实体类型 | 11类 |
| 格式 | UTF-8 JSONL，字符级BIOS标注 |
| 嵌套实体 | **不支持**（明确标注 no nested entities） |

### 实体类型（11类）

| 缩写 | 全称 | 说明 |
|---|---|---|
| TCMDM | TCM Disease Term | 中医疾病术语 |
| WMDT | Western Medicine Disease Term | 西医疾病术语 |
| TCMSY | TCM Syndrome | 中医证候 |
| UD | Unknown Disease | 未明确疾病 |
| PC | Pathological Change | 病理变化 |
| TC | Treatment Concept | 治疗概念 |
| WMS | Western Medicine Symptom | 西医症状 |
| TCMS | TCM Symptom | 中医症状 |
| TCMTM | TCM Treatment Method | 中医治法 |
| TCMP | TCM Prescription | 中医处方 |
| CMM | Chinese Materia Medica | 中药 |

### 数据格式示例

```json
{"text": "黄", "label": "B-CMM"}
{"text": "芪", "label": "I-CMM"}
{"text": "", "label": ""}  // 文档分隔符
```

### 获取方式

- **Demo子集**：IEEE DataPort 直接下载（train_demo.jsonl / dev_demo.jsonl / test_demo.jsonl）
- **完整版**：邮件申请（tongyu@stu.edu.cn / chpcai@stu.edu.cn）
  - 需提供：研究计划 + 学术用途证明 + 隐私保护承诺 + 引用承诺
  - 学术免费

### Baseline结果

论文中报告了BiLSTM-CRF、BERT、RoBERTa、SLGAN + LLM的baseline结果。

### 评价

- **优点**：规模大、标注质量高（双盲标注+专家仲裁）、实体类型丰富、格式标准
- **缺点**：**不支持嵌套实体**，与我们创新点（融合KG先验解决嵌套问题）有矛盾
- **适用**：非嵌套NER训练可用，嵌套NER实验做不了

## 二、ShenNong_TCM_Dataset（HuggingFace）

### 基本信息

| 项目 | 内容 |
|---|---|
| 来源 | 华东理工大学 |
| HuggingFace | michaelwzhu/ShenNong_TCM_Dataset |
| 规模 | 113,000 QA对 |
| 格式 | 问答对（query + response） |
| 用途 | LLM指令微调 |

### 数据示例

```
Q: 我腹痛，没有其他症状，有什么中药可以推荐吗？
A: 首先需要确定腹痛的性质（寒热虚实），进而确定证候...
```

### 评价

- **不适合我们**：这是QA数据集，不是NER/RE标注数据
- 可作为LLM辅助生成训练数据时的参考模板

## 四、TCM_KG（GitHub）

### 基本信息

| 项目 | 内容 |
|---|---|
| GitHub | Zescy1/TCM_KG |
| 更新时间 | 2026-01-16 |
| 规模 | ~17,000 个TCM实体 |
| 格式 | JSON |

### 数据来源

| 子目录 | 内容 |
|---|---|
| 1_SymMap/ | 症状映射数据（Herbs, Symptoms） |
| 2_Ali/ | 阿里处方数据 |
| 3_20/ | 20味中药数据集 |
| 4_book/ | 中医古籍提取 |
| 5_君佐陈使/ | 方剂配伍原则 |
| 6_妇科同义词/ | 妇科同义词 |
| 7_方剂/ | 方剂数据 |
| 8_专业词汇--QA/ | 专业词汇+QA对 |

### 节点与关系

- **节点类型**：中药、症状、疾病、方剂
- **关系类型**：treats（治疗）、composed_of（组成）、classified_as（归经）等
- **词表**：vocab.json，过滤规则（无英文/无括号/长度≤10）

### 评价

- **可作为KG先验知识来源**：提取配伍关系、药症关系等特征
- 直接可用，无需申请

## 五、TCMSP — 药理学特征数据源

### 基本信息

| 项目 | 内容 |
|---|---|
| 全称 | Traditional Chinese Medicine Systems Pharmacology |
| 论文 | PMC4001360（2014） |
| 内容 | 中药化学成分 + 药理学参数 |

### 药理学参数

| 参数 | 说明 | 用途 |
|---|---|---|
| OB | 口服生物利用度 | 药物吸收能力 |
| DL | 类药性 | 成药可能性 |
| MW | 分子量 | 分子大小 |
| AlogP | 脂溶性 | 亲脂性 |
| Hdon | 氢键供体 | 分子间作用力 |
| Hacc | 氢键受体 | 分子间作用力 |

### 获取方式

- 网站直接查询
- **TCMSP-Exporter**（GitHub: shujuecn/TCMSP-Exporter）：浏览器端批量抓取，支持OB/DL过滤，导出XLSX

### 评价

- **核心数据源**：我们药理学特征融合层的数据来源
- 完全可用，无需申请

## 七、综合评估

### 需求 vs 数据源对照

| 我们的需求 | 最佳数据源 | 状态 | 备选 |
|---|---|---|---|
| NER训练（嵌套） | DG-SpanTCM自建数据集 | 未公开 | TCMNER2026（非嵌套） |
| NER训练（非嵌套） | TCMNER2026 | Demo可用，完整版需申请 | — |
| RE训练 | 无现成数据集 | 需自建 | TCM_KG三元组+LLM生成 |

## 八、最终数据方案（2026-07-22定稿）

> TCMNER2026被拒 + DG-SpanTCM未回复 → 调整为自建数据方案

### 核心思路

**不依赖任何外部NER数据集**，采用"词典匹配+LLM标注+人工审核"构建TCM-NER-3000，同时用LLM辅助构建TCM-RE-3000。

### 数据构建流程

```
中医公开文本（古籍、教材、药典）
        │
        ├── TCM_KG词典匹配预标注
        │
        ├── LLM自动标注（GPT/ChatGLM）
        │
        └──────────┬──────────
                   │
            人工审核（30-50%）
                   │
            TCM-NER-3000
                   │
        BERT+SoftLexicon+CRF
                   │
              NER识别结果
                   │
            实体两两配对
                   │
           LLM判断关系+Self-consistency
                   │
            人工审核（20%）
                   │
             TCM-RE-3000
                   │
             RE模型训练
```

### 数据规模估算

| 数据集 | 规模 | 内容 | 工作量 |
|---|---|---|---|
| TCM-NER-3000 | 3000句 | ≈25000-35000实体，6类 | 词典匹配+LLM标注2-3天，人工审核30-50%约3-5天 |
| TCM-RE-3000 | 3000条 | 5类关系+no_relation | LLM标注2-3天，人工审核20%约1-2天 |

### 数据来源

| 来源 | 内容 | 用途 |
|---|---|---|
| TCM_KG | 实体词典(~17,000词) + 三元组 | NER词典匹配 + RE种子知识 |
| 中医公开文本 | 古籍、教材、药典 | NER原始语料 |
| TCMNER2026 Demo | 少量中医病历 | 外部验证（不作为主数据） |

### 创新点调整

从"模型创新"调整为**"数据+模型协同创新"**：

1. **创新一（新增）**：提出基于LLM的中医NER/RE数据自动构建方法
2. **创新二（保留）**：SoftLexicon词典增强的中医NER
3. **创新三（保留）**：药理特征+KG先验融合的RE

### 论文表述

"针对中医领域标注数据稀缺的问题，提出一种基于LLM的低成本数据构建流程：利用TCM_KG词典进行预标注，结合LLM自动标注与人工审核，构建TCM-NER-3000和TCM-RE-3000数据集。该方法可复现、可扩展，为中医NLP研究提供了数据构建范式。"
