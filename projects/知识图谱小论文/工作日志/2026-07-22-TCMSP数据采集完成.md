# TCMSP药理学数据采集 — 完成报告

## 一、背景

按照[[07-技术架构-完整版]]中的数据方案，需要从TCMSP（中药系统药理学数据库）爬取中药的成分、靶点、疾病数据，作为RE模型的药理学特征来源。

## 二、采集过程

### 2.1 工具选型

| 方案 | 结果 |
|---|---|
| TCMSP-Exporter浏览器插件 | Failed to fetch，无法连接 |
| TCMSP-Spider（GitHub） | 已归档，页面解析规则过期 |
| 自写Python爬虫 | 采用，基于TCMSP-Exporter的API逻辑 |

### 2.2 爬虫实现

核心API模式：
```
搜索中药名: {root_url}?qs=herb_all_name&q={中文名}&token={token}
获取详情:   {root_url}?qr={英文名}&qsr=herb_en_name&token={token}
```

数据从页面JavaScript的kendoGrid初始化代码中提取，用正则匹配`data:`后面的JSON数组。

### 2.3 批量处理

- 输入：1720味中药（来自TCM_KG的node数据，type='中药'）
- 分批：每批100个，批次间休息30秒
- Token刷新：每批开始前重新获取
- 请求间隔：2-4秒随机延迟

### 2.4 失败分析

| 类别 | 数量 | 说明 |
|---|---|---|
| 成功 | 530 | 有成分/靶点/疾病数据 |
| 剂量描述 | 315 | 古籍用量描述，如"杏仁一升，去皮尖" |
| 辅料 | 99 | 化工辅料，如"氮酮"、"甘油" |
| 其他 | 776 | 不在TCMSP数据库中的药材 |

### 2.5 重试验证

对776个"其他"类进行了重试验证（抽样230个），**全部失败**。原因：

- **炮制品**：制何首乌、侧柏炭、杜仲炭 → TCMSP只收录原药材
- **动物药**：乌鸡、兔睾丸、僵蚕 → 不在TCMSP收录范围
- **极冷门药材**：万筋藤、假耧斗菜 → TCMSP未收录
- **制剂/辅料**：乳糖、冰糖、乌鸡白凤丸 → 非单味药

验证方法：对矮地茶（成功）和朱砂（失败）进行API对比测试，确认是数据库本身不收录，非爬虫问题。

## 三、最终数据

### 3.1 数据规模

| 指标 | 数值 |
|---|---|
| Excel文件数 | 496个 |
| 唯一英文名 | 494个（2个有重名覆盖） |
| 中文名→英文名映射 | 530个中文名 → 494个英文名 |

差异说明：87个英文名被多个中文名共享，如：
- 胡黄连/黄连/连/黄 → Picrorhizae Rhizoma
- 熟地黄/地黄/黄/地 → Rehmanniae Radix Praeparata

### 3.2 数据结构

每个Excel文件包含三个Sheet：

| Sheet | 内容 | 示例字段 |
|---|---|---|
| Ingredients | 化学成分 | molecule_name, MW, AlogP, Hdon, Hacc, OB, DL |
| Targets | 靶点蛋白 | target_name, UniProt_id |
| Diseases | 关联疾病 | disease_name |

### 3.3 数据质量

- 496个Excel文件均有实际数据，无空文件
- 无重复文件
- 无保存错误
- 覆盖常见中药材，对KG构建足够

## 四、数据用途

根据[[07-技术架构-完整版]]的设计，TCMSP数据用于：

1. **RE模型药理学特征**：OB、DL、MW、AlogP、Hdon、Hacc 6维特征向量
2. **实体字典补充**：成分名（Component）和疾病名（Disease）作为NER词典
3. **关系验证**：herb-component、component-target、component-disease 三类关系
