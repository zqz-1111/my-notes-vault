# NER数据构建 — 进行中

## 一、今日工作

### 1.1 语料抽取

从704本中医古籍中抽取句子，使用Aho-Corasick算法做高效实体匹配：

| 指标 | 数值 |
|---|---|
| 古籍数 | 701本（3本读取失败） |
| 总句子数 | 3,505,362 |
| 去重后 | 3,155,846 |
| 含实体句子 | 1,804,460 |
| 去重后含实体 | 1,651,580 |

实体匹配使用TCM_KG词典（16,383个实体），覆盖率达48.5%。

### 1.2 LLM标注V1

从165万句中分层抽样3,000句，用DeepSeek API标注：

- 标注完成：3,000句全部标完
- 有实体：2,705句（90.2%）
- 总实体数：10,327个
- 平均每句：3.4个实体

**抽检结果**：错误率约30%，主要问题：
- 书名误标为方剂（《内经》→ Formula）
- 身体部位误标为症状（胸中 → Symptom）
- 用法误标为中药（生用 → Herb）
- 治法误标为方剂（大补血气 → Formula）
- 生理概念误标为疾病（阳气 → Disease）

### 1.3 规则清洗

基于黑名单+正则规则清洗V1标注结果：

| 指标 | 清洗前 | 清洗后 |
|---|---|---|
| 实体数 | 9,996 | 8,620 |
| 删除 | - | 1,376（13.8%） |

清洗后抽检错误率：16%（从30%降到16%）。

### 1.4 LLM标注V2（进行中）

改进Prompt后重新标注3,000句：
- 每个实体类型加了正例/反例
- 增加6条标注规则（书名不标、剂型泛指不标、治法不标等）
- 输出格式简化（去掉位置信息）
- 预期错误率降至10-15%
