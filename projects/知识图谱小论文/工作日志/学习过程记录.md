### 三、技术学习：NER 核心机制

通过阅读 bert-crf-chinese-ner 项目代码，掌握了以下核心知识：

#### BIO 标注体系
- B = Begin（实体开始），I = Inside（实体内部），O = Outside（非实体）
- 识别规则：B-X 开头 + 连续 I-X → 提取为类型 X 的实体
- 示例：
  ```
  黄  芪  是  补  气  药
  B-Her I-Her O  O  O  O
  ```

#### CRF 层的作用
- 问题：BERT + Softmax 预测每个字符标签是独立的，容易产生非法转移（如 O 后面跟 I-Her）
- 解决：CRF 学习一个转移矩阵 T(K×K)，显式建模标签间的合法转移
- 效果：提升约 1-1.5 个 F1 百分点

#### BERT + CRF 数据流
```
输入汉字序列
    ↓
BERT编码 → 768维向量
    ↓
线性层 → 映射到K维（发射分数）
    ↓
CRF层 → 训练时算负对数似然Loss / 推理时Viterbi解码最优序列
    ↓
后处理 → 提取实体
```

#### 工程踩坑
1. **CRF库不支持-100**：PyTorch用-100表示忽略位，但torchcrf不认识，需临时替换为0
2. **加载未微调的BERT**：save_pretrained只存BERT部分，预测时需强制加载微调后的权重
3. **长文本截断越界**：Tokenizer截断后仍用原始长度切片，需取min(len(chars), len(indices))

#### 论文12：DG-SpanTCM（Nature Scientific Reports, 2025）

- **标题**：Nested NER in TCM EMRs via Dual-Granularity Feature Augmentation and Span Classification
- **核心方案**：BERT + SoftLexicon + 对抗训练(FGM) + Span分类 + ACW Loss
- **关键数据**：F1=70.53%（自建TCM数据集），70.51%（CMeEE）

**核心机制：**

1. **双粒度编码**：BERT（字级）+ SoftLexicon（词级BMES）→ Concat+LayerNorm
   - SoftLexicon用中医词典匹配，为每个字构建B/M/E/S四元组
   - TF-IDF权重衰减：高频词权重降低，罕见专业词放大
   - 不用jieba的原因：硬切分会破坏嵌套结构，错误传播

2. **对抗训练FGM**：在Embedding层添加微小扰动，增强鲁棒性
   - 消融实验证明影响最大（-3.96% F1）

3. **Span分类解码**：抛弃CRF，枚举所有可能的span(i,j)
   - n个字 → n(n+1)/2个span
   - 解码矩阵W(n×n上三角)，每个元素存储预测类别
   - 乘法注意力捕捉span内部依赖，MLP分类

4. **ACW Loss**：根据类别频率动态调整权重，解决长尾分布
   - γ参数控制调整剧烈程度
   - 配合噪声感知训练策略

**数据集**：自建42,500句/43,028实体/17类/嵌套率52.3%，来自江西中医药大学2014-2021

**消融实验结论**：
- 去掉对抗训练 → F1-3.96%（影响最大）
- 去掉词典匹配 → F1-2.71%
- ACW→交叉熵 → F1-2.63%

### 二、论文13精读：TBTC知识蒸馏

**核心方案**：Teacher（简单结构+高密度数据）→ Student（TBTC复杂结构+稀疏数据）

**TBTC四层架构**：
1. TCMBERT：中医语料预训练，领域语义编码
2. BiLSTM：局部序列依赖建模
3. Transformer（4层）：长距离全局语义建模
4. CRF：标签转移约束+边界解码

**蒸馏机制**：
- Teacher的Pre-CRF logits → 温度缩放Softmax（τ=5）→ 软标签
- Student同时受真实标签和软标签监督
- KL散度度量师生分布差异
- 联合损失：Z_total = α × Z_KD + (1-α) × Z_CRF，最优α=0.9

**关键发现**：
- Teacher核心优势在数据质量而非模型复杂度
- 蒸馏后F1=77.01%，提升6.67%
- 低频实体（中药性能）F1净增26.62%
- 消融：去掉KD模块F1骤降6.67%（影响最大）

**三条技术路线对比**：
1. 经典蒸馏：有结构化辞典时首选，成本适中
2. LLM生成数据：什么都没有时用，成本高有幻觉风险
3. 蒸馏+LLM结合：最优但成本最高

**我们的情况**：有TCMSP结构化数据 → 经典蒸馏是首选

**核心发现**：Fine-tuned GPT-3.5（F1=0.92）击败GPT-4

**原因**：领域规则记忆 > 通用推理能力

**Prompt设计**：结构化约束（实体列表+关系定义+输出格式），把LLM当"带约束的关系分类器"

**对我们的启发**：
- 领域小任务：微调小模型 > 通用大模型
- 最佳方案：NER + LLM RE混合架构
  - NER精准找实体
  - LLM灵活判断关系（给实体列表+关系定义+格式约束）
