# 知识图谱小论文 — 执行计划

## 一、项目定位

| 项目 | 内容 |
|---|---|
| 选题 | 中药方剂领域知识图谱构建 + 融合药理学特征的关系抽取改进 |
| 目标 | EI 会议（CA 检索），4-6 页 |
| 创新点 | 在 BERT 基础上，用注意力机制融合 OB/DL 药理学特征提升关系抽取准确率 |
| 时间投入 | 每天 2-3 小时 |
| 技术基础 | Python 会用，NLP 课上过但不够扎实，需要边做边学 |
| 策略 | 不急，做好再投。先补基础再动手，确保每一步都理解 |

## 二、16 周执行计划（含学习缓冲）

### 第一阶段：文献调研 + NLP 补课（第 1-4 周）

**目标**：读完核心文献，补上 NLP/NER/RE 的实操能力

#### 第 1 周：文献综述 + 领域全貌

| 任务 | 具体内容 | 产出 |
|---|---|---|
| 读论文1 | TCM KG 综述（2024），已读，回顾笔记 | 综述笔记 |
| 读论文15 | ML + LLM 混合构建 KG（2025） | 笔记 |
| 读论文12 | 嵌套 NER（Nature 2025） | 笔记 |
| 整理文献对比表 | 每篇论文的问题/方法/结果/局限 | 文献对比表 |

#### 第 2 周：NER/RE 技术文献 + 基础补课

| 任务 | 具体内容 | 产出 |
|---|---|---|
| 读论文7 | BERT-BiLSTM-CRF 经典方法 | 笔记 |
| 读论文13 | 知识蒸馏 NER（Nature 2026） | 笔记 |
| 补课 | 跑通一个中文 NER demo（用 HuggingFace + BERT） | 可运行的 NER 代码 |
| 补课 | 理解 BIO 标注体系、CRF 层原理 | 学习笔记 |

#### 第 3 周：RE 技术补课 + 数据集调研

| 任务 | 具体内容 | 产出 |
|---|---|---|
| 读论文8 | GPT 做关系抽取（2024） | 笔记 |
| 读论文14 | R1-RE 跨域 RE（2026） | 笔记 |
| 补课 | 跑通一个中文 RE demo（BERT 做关系分类） | 可运行的 RE 代码 |
| 调研数据集 | 确认 TCMNER2026、TCMSP、ShenNong 数据集可用性 | 数据集清单 |

#### 第 4 周：创新点固化 + 技术方案定稿

| 任务 | 具体内容 | 产出 |
|---|---|---|
| 整理文献笔记 | 综合所有文献，找出研究缺口 | 文献综述初稿（2000字） |
| 固化创新点 | 明确"融合药理学特征的 BERT RE"的技术细节 | 创新点说明书 |
| 画技术架构图 | 整体框架、各模块、数据流 | 架构图 |
| 和导师沟通 | 汇报选题和方案，获取认可 | 导师反馈 |

**阶段验收**：文献综述初稿 + 创新点说明书 + 导师认可

### 第二阶段：数据收集与处理（第 5-7 周）

**目标**：拿到干净的、可训练的数据

#### 第 5 周：TCMSP 数据获取

| 任务 | 具体内容 | 产出 |
|---|---|---|
| TCMSP 数据爬取 | 写 Python 脚本爬取中药-成分-靶点-疾病数据 | 原始数据文件 |
| 数据清洗 | 去重、格式统一、缺失值处理 | 清洗后数据 |
| 数据探索 | 统计实体数量、关系分布、数据质量 | 数据分析报告 |

#### 第 6 周：NER/RE 标注数据准备

| 任务 | 具体内容 | 产出 |
|---|---|---|
| 下载公开数据集 | TCMNER2026、ShenNong_TCM_Dataset | 数据文件 |
| 数据格式转换 | 统一为训练格式（BIO 标注、关系对） | 格式化数据 |
| 补充标注（如需要） | 用 Label Studio 标注额外数据 | 标注数据 |

#### 第 7 周：数据集划分 + 基线统计

| 任务 | 具体内容 | 产出 |
|---|---|---|
| 数据集划分 | 训练集/验证集/测试集（8:1:1） | 划分后数据 |
| 基线统计 | 实体类型分布、关系类型分布、样本长度分布 | 统计图表 |
| 药理学特征提取 | 从 TCMSP 提取 OB/DL 数值，关联到实体对 | 特征文件 |

**阶段验收**：干净的训练数据 + 药理学特征文件 + 数据分析报告

## 三、每周检查清单

每周日对照检查，记录进度：

```
□ 本周计划任务是否完成？
□ 产出物是否到位？
□ 遇到什么问题？如何解决？
□ 下周计划是否需要调整？
□ 是否需要和导师沟通？
```

## 四、关键里程碑

| 里程碑 | 预计时间 | 验收标准 |
|---|---|---|
| M1：文献调研完成 | 第 4 周末 | 文献综述初稿 + 创新点说明书 + 导师认可 |
| M2：数据就绪 | 第 7 周末 | 干净数据 + 特征文件 + 数据分析报告 |
| M3：实验完成 | 第 12 周末 | 完整实验数据 + 消融实验 + 知识图谱 |
| M4：论文终稿 | 第 15 周末 | 完整论文 + 导师认可 |
| M5：投稿完成 | 第 16 周末 | 投稿确认邮件 |

## 五、风险与应对

| 风险 | 概率 | 应对 |
|---|---|---|
| 数据质量差 | 中 | 第 5-6 周提前发现，换数据源或补充标注 |
| 改进效果不明显 | 中 | 调整融合策略，或换更简单的改进点 |
| 实验跑不通 | 低 | 参考 GitHub 开源代码，AI 辅助 debug |
| 导师要求大改 | 中 | 每个里程碑都和导师沟通，避免最后才发现 |
| 会议截稿赶不上 | 低 | 不急，做好再投，有多个会议可选 |
