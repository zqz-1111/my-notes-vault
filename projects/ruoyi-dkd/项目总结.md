# ruoyi-dkd 项目总结

## 项目简介

帝可得（dkd）智能售货机运营管理系统，基于若依（RuoYi-Vue）后台管理框架，前后端分离，Vue 3 + Vite + Element Plus + Spring Boot + MyBatis。

## 技术栈

| 层 | 技术 |
|---|---|
| 前端 | Vue 3、Vite、Element Plus、Pinia、axios |
| 后端 | Spring Boot、MyBatis、Redis、MinIO、EasyExcel |
| 数据库 | MySQL |
| 部署 | 前后端同一 Git 仓库 |

## 前端改造核心模式

| 模式 | 说明 |
|---|---|
| 搜索精简 | 去掉多余搜索字段，只保留关键的 1-2 个 |
| 序号列 | `type="index"` 自动生成递增序号 |
| 下拉选择 | `el-select` + `v-for` 替代输入框（区域、角色、类型等） |
| 嵌套对象 | `prop="region.regionName"` 展示后端嵌套对象 |
| v-for 匹配 | 后端只返回 ID 时，前端循环匹配显示名称 |
| 条件渲染 | `v-if="form.id!=null"` 区分新增/编辑模式 |
| 价格转换 | 存分显示元：`/100` 展示，`Math.round(*100)` 保存 |
| 字典组件 | `<dict-tag>` 显示状态、类型等字典字段 |
| 查看详情 | `el-descriptions` 或子表格展示关联数据 |
| 父子组件通信 | `defineProps` + `defineEmits` + `watch` |
| Excel 导入 | `el-upload` 手动上传 + Token 鉴权 |

## 踩坑汇总

| 坑 | 说明 |
|---|---|
| JS 注释在 `<style>` 标签上无效 | `//` 是 JS 注释，放在 `</script>` 后面不生效，用 `<!-- -->` |
| scoped 样式不影响子组件 | 父组件不带 scoped 的样式子组件可继承，但子组件应自己引入 |
| 浮点精度问题 | `0.1 * 100 ≠ 10`，用 `Math.round()` |
| ref 初始化类型 | API 返回数组就初始化为 `[]`，返回对象就初始化为 `{}` |
| async 无 try-catch 导致 loading 卡死 | 用 `try-catch-finally` 确保 loading 关闭 |
| API 路径参数 vs 查询参数 | 后端 `@GetMapping("/list")` 没有路径变量，前端用 `params` 传参 |
| MySQL 保留字 | `desc` 等字段名需要加反引号 |
| Long 比较用 equals | Java 中 `==` 比较引用，`equals()` 比较值 |
| Lombok @Data 不生效 | 手写 getter/setter 更可靠 |
