# 后端概述

## 一、项目简介

**项目名称**：帝可得管理系统（DKD Management System）
**版本**：3.8.7
**基础框架**：基于 RuoYi 脚手架（前后端分离版）
**业务场景**：智能售货机平台管理 —— 设备部署、补货、维修、撤机等 B2B 运维工单管理

### 4.2 dkd-common — 公共基础模块

零 Spring Web 依赖，所有模块的底层基石。

**自定义注解**：

| 注解 | 用途 |
|---|---|
| `@Log` | 操作日志记录（审计） |
| `@DataScope` | 数据权限过滤（部门级数据隔离） |
| `@DataSource` | 多数据源切换（MASTER/SLAVE） |
| `@Excel` / `@Excels` | Excel 导入导出字段映射 |
| `@RateLimiter` | Redis 限流 |
| `@RepeatSubmit` | 防重复提交 |
| `@Anonymous` | 匿名访问接口 |
| `@Sensitive` | 数据脱敏（手机号、邮箱、身份证、银行卡） |

**核心类**：

| 类 | 说明 |
|---|---|
| `BaseEntity` | 基础实体（createBy、createTime、updateBy、updateTime、remark） |
| `TreeEntity` | 树形实体（extends BaseEntity，加 parentId、children） |
| `AjaxResult` | 统一响应体（code、msg、data） |
| `LoginUser` | 实现 UserDetails，持有用户信息 + Token + 权限集合 |
| `RedisCache` | Redis 工具类（String/List/Set/Map 操作、TTL 管理） |
| `BaseController` | 基础 Controller（分页、响应工具方法） |
| `SecurityUtils` | 获取当前登录用户、用户ID、密码加密 |

**业务常量**（`DkdContants`）：
- 工单类型：部署(1)、补货(2)、维修(3)、撤机(4)
- 工单状态：创建(1)、进行中(2)、取消(3)、完成(4)
- 售货机状态：未部署(0)、运行中(1)、已撤机(3)
- 角色编码：运营(1002)、维修(1003)

**异常体系**：`ServiceException`（业务异常）、`GlobalException`、各种文件/用户/验证码异常

### 4.3 dkd-framework — 框架核心层

把 Security、AOP、配置全部串起来。

**AOP 切面**：

| 切面 | 拦截注解 | 功能 |
|---|---|---|
| `LogAspect` | `@Log` | 记录操作日志（请求/响应/IP/耗时） |
| `DataScopeAspect` | `@DataScope` | 追加 SQL 条件实现行级数据权限 |
| `DataSourceAspect` | `@DataSource` | 切换主从数据源 |
| `RateLimiterAspect` | `@RateLimiter` | Redis Lua 脚本限流 |

**配置类**：

| 类 | 功能 |
|---|---|
| `SecurityConfig` | Spring Security：无状态会话（JWT）、BCrypt、CORS、放行 URL |
| `RedisConfig` | RedisTemplate + FastJson2 序列化 + 限流 Lua 脚本 Bean |
| `MyBatisConfig` | SqlSessionFactory 通配符扫描 |
| `DruidConfig` | 主从多数据源 + DynamicDataSource |
| `CaptchaConfig` | 验证码（数学/字符两种） |
| `FilterConfig` | XSS 过滤器 + 可重复读过滤器 |
| `ThreadPoolConfig` | 线程池（core=50, max=200, queue=1000） |

**Security 流程**：
```
登录请求 → SysLoginService（验证码校验 → Spring Security 认证）
  → UserDetailsServiceImpl（查用户 → 返回 LoginUser）
  → TokenService（生成 JWT → 存 Redis → 返回 Token）

后续请求 → JwtAuthenticationTokenFilter（解析 Token → 从 Redis 取 LoginUser → 设 SecurityContext）
```

**权限控制**：`@PreAuthorize("@ss.hasPermi('system:user:list')")`  —— 自定义 `PermissionService` Bean 做权限判断

**核心 Service**：

| 类 | 功能 |
|---|---|
| `TokenService` | JWT 创建（HS512）、解析、自动续期（<20 分钟时刷新） |
| `SysLoginService` | 登录全流程（验证码 → 预检查 → 认证 → 生成 Token） |
| `SysPasswordService` | 密码校验 + 重试限制（5 次锁定 10 分钟） |
| `PermissionService` | hasPermi / hasRole 权限判断（SpEL 表达式调用） |

### 4.4 dkd-system — 系统业务层

系统管理的实体、Mapper、Service 实现。

**实体类**：`SysUser`、`SysRole`、`SysMenu`、`SysDept`、`SysConfig`、`SysDictData`、`SysDictType`、`SysLogininfor`、`SysOperLog`、`SysNotice`、`SysPost` 以及各种关联表（`SysUserRole`、`SysRoleMenu`、`SysRoleDept`、`SysUserPost`）

**Mapper XML**：`dkd-system/src/main/resources/mapper/system/` 下一个 XML 对应一个 Mapper

**路由生成**：`ISysMenuService.selectMenuTree()` → 构建前端路由树（`RouterVo` + `MetaVo`）

### 4.5 dkd-quartz — 定时任务

基于 Quartz 的动态任务管理，支持 Web 界面增删改查。

**核心类**：
- `AbstractQuartzJob` — 模板方法模式（before → doExecute → after），记录执行日志
- `QuartzJobExecution` — 并发执行实现
- `QuartzDisallowConcurrentExecution` — 非并发执行
- `JobInvokeUtil` — 反射调用 Spring Bean 方法（通过 `invokeTarget` 字符串表达式）
- `RyTask` — 示例任务 Bean（无参/单参/多参方法）

**注意**：`ScheduleConfig` 中数据库持久化配置被注释掉了，当前单机模式用 RAM 存储。

### 4.6 dkd-generator — 代码生成

从数据库表结构自动生成全套 CRUD 代码。

## 五、安全架构

- **认证**：JWT 无状态认证，Token 存 Redis，默认 30 分钟过期，<20 分钟自动续期
- **密码**：BCrypt 哈希，5 次错误锁定 10 分钟
- **授权**：`@PreAuthorize("@ss.hasPermi('...')")` 基于 SpEL 的权限校验
- **数据权限**：`@DataScope` AOP 追加 WHERE 条件（全部/自定义/部门/部门及下级/仅本人）
- **防 XSS**：过滤器 + `@Xss` 注解
- **限流**：`@RateLimiter` Redis Lua 脚本
- **防重提交**：`@RepeatSubmit` Redis 记录请求指纹
- **IP 黑名单**：通过 `sys.login.blackIPList` 系统配置

## 六、Redis 使用场景

| Key 模式 | 用途 | TTL |
|---|---|---|
| `login_tokens:{uuid}` | 存储 LoginUser 对象 | 30 分钟 |
| `captcha_codes:{uuid}` | 验证码文本 | 2 分钟 |
| `sys_config:{key}` | 系统配置缓存 | — |
| `sys_dict:{dictType}` | 字典数据缓存 | — |
| `rate_limit:{key}` | 限流计数器 | 配置 |
| `pwd_err_cnt:{username}` | 密码错误次数 | 10 分钟 |
| `repeat_submit:{key}` | 重复提交指纹 | 5 秒 |
