# 【业务逻辑】新增设备自动生成货道 + 修改设备同步冗余字段

## 一、写在前面

1. **解决什么问题**：新增售货机时，需要根据设备类型的「行×列」自动生成所有货道记录，同时从点位表同步区域、合作商等冗余字段到设备表。
2. **学完你能掌握**：Service 层多表操作、@Transactional 事务控制、批量插入、冗余字段同步。

## 二、业务流程

新增设备时要做 3 件事：

1. **补充设备信息**：生成编号、查 VmType 补容量、查 Node 补区域/合作商/地址
2. **保存设备**：写入 tb_vending_machine
3. **自动生成货道**：根据 VmType 的 vmRow × vmCol，批量插入 tb_channel

## 三、代码实现

### 3.1 insertVendingMachine — 新增设备

```java
@Transactional
@Override
public int insertVendingMachine(VendingMachine vendingMachine) {
    // 1. 生成8位编号
    String innerCode = UUIDUtils.getUUID();
    vendingMachine.setInnerCode(innerCode);

    // 2. 查设备类型，补充容量
    VmType vmType = vmTypeService.selectVmTypeById(vendingMachine.getVmTypeId());
    vendingMachine.setChannelMaxCapacity(vmType.getChannelMaxCapacity());

    // 3. 查点位表，补充冗余字段（区域、合作商、地址、商圈）
    Node node = nodeService.selectNodeById(vendingMachine.getNodeId());
    BeanUtil.copyProperties(node, vendingMachine, "id"); // 复制除id外的所有属性
    vendingMachine.setAddr(node.getAddress());

    // 4. 设置状态和时间
    vendingMachine.setVmStatus(DkdContants.VM_STATUS_NODEPLOY);
    vendingMachine.setCreateTime(DateUtils.getNowDate());
    vendingMachine.setUpdateTime(DateUtils.getNowDate());

    // 5. 保存设备
    int result = vendingMachineMapper.insertVendingMachine(vendingMachine);

    // 6. 自动生成货道（双层for循环）
    List<Channel> channelList = new ArrayList<>();
    for (int i = 1; i <= vmType.getVmRow(); i++) {
        for (int j = 1; j <= vmType.getVmCol(); j++) {
            Channel channel = new Channel();
            channel.setChannelCode(i + "-" + j);
            channel.setVmId(vendingMachine.getId());
            channel.setInnerCode(vendingMachine.getInnerCode());
            channel.setMaxCapacity(vmType.getChannelMaxCapacity());
            channel.setCreateTime(DateUtils.getNowDate());
            channel.setUpdateTime(DateUtils.getNowDate());
            channelList.add(channel);
        }
    }

    // 7. 批量插入货道
    channelService.batchInsertChannel(channelList);
    return result;
}
```

### 3.2 updateVendingMachine — 修改设备

```java
@Override
public int updateVendingMachine(VendingMachine vendingMachine) {
    // 修改时也要同步点位的冗余字段
    Node node = nodeService.selectNodeById(vendingMachine.getNodeId());
    BeanUtil.copyProperties(node, vendingMachine, "id");
    vendingMachine.setAddr(node.getAddress());
    vendingMachine.setUpdateTime(DateUtils.getNowDate());
    return vendingMachineMapper.updateVendingMachine(vendingMachine);
}
```

### 3.3 批量插入货道 — batchInsertChannel

**ChannelMapper.java**：

```java
public int batchInsertChannel(List<Channel> channelList);
```

**ChannelMapper.xml**：

```xml
<insert id="batchInsertChannel" parameterType="java.util.List">
    INSERT INTO tb_channel (
        channel_code, vm_id, inner_code, max_capacity,
        last_supply_time, create_time, update_time
    ) VALUES
    <foreach collection="list" item="channel" separator=",">
        (
            #{channel.channelCode},
            #{channel.vmId},
            #{channel.innerCode},
            #{channel.maxCapacity},
            #{channel.lastSupplyTime},
            #{channel.createTime},
            #{channel.updateTime}
        )
    </foreach>
</insert>
```

## 四、踩坑避坑

### 逻辑坑：BeanUtil.copyProperties 的第三个参数

- **问题**：`BeanUtil.copyProperties(node, vendingMachine, "id")` 会把 node 的所有属性复制到 vendingMachine，包括 id
- **后果**：设备的 id 会被点位的 id 覆盖，导致插入失败
- **解决**：第三个参数 `"id"` 是忽略的字段名，表示不复制 id

### 逻辑坑：@Transactional 的必要性

- **问题**：新增设备和新增货道是两步操作，如果货道插入失败，设备已经插入了
- **后果**：数据库里有设备但没有货道，数据不一致
- **解决**：加 `@Transactional` 注解，任何一步失败都会回滚

## 六、总结

- 新增设备是一个「多表联动」的业务：设备表 + 货道表 + 点位表 + 设备类型表
- `BeanUtil.copyProperties` 可以快速复制对象属性，但要注意排除 id 等不该复制的字段
- 批量插入用 MyBatis 的 `<foreach>` 标签，比循环单条插入高效
- 涉及多表写操作时必须加 `@Transactional` 保证数据一致性
