## 二、踩坑修复

### 2.1 MySQL 保留字问题（9.2.3）

**问题：** 工单表的 `desc` 字段是 MySQL 保留字（用于排序 `ORDER BY xxx DESC`），导致 SQL 语法错误

**解决方案：** 给所有 `desc` 字段加反引号

```xml
<!-- ❌ 错误：MySQL 把 DESC 解析成排序关键字 -->
select task_id, desc from tb_task

<!-- ✅ 正确：反引号告诉 MySQL 这是字段名 -->
select task_id, `desc` from tb_task
```

**需要修改的 4 处：**
1. `selectTaskVo` 中的字段名
2. `selectTaskList` 中的查询条件
3. `insertTask` 中的字段名
4. `updateTask` 中的字段名

**常见保留字：** `DESC`、`ORDER`、`GROUP`、`KEY`、`INDEX`、`STATUS`、`USER`、`TYPE`、`NAME`

**最佳实践：** 建表时尽量避免使用保留字，比如用 `description` 代替 `desc`

## 三、TaskVo 封装（9.3.2）

### 3.1 为什么需要 TaskVo？

工单列表需要显示**工单类型名称**，但工单表只存了 `productTypeId`（类型ID），需要关联查询工单类型表。

**Vo（View Object）作用：** 扩展实体类，添加关联数据

```java
public class TaskVo extends Task {
    // 工单类型
    private TaskType taskType;
    // 手写 getter/setter（避免 Lombok 版本冲突）
}
```

### 3.2 Lombok 版本冲突问题

项目使用的 JDK 1.8 与 Lombok 高版本不兼容，`@Data` 注解不生效。

**解决方案：** Vo 类手写 getter/setter

```java
// ❌ 不生效
@Data
public class TaskVo extends Task {
    private TaskType taskType;
}

// ✅ 手写
public class TaskVo extends Task {
    private TaskType taskType;
    
    public TaskType getTaskType() {
        return taskType;
    }
    
    public void setTaskType(TaskType taskType) {
        this.taskType = taskType;
    }
}
```

## 四、关联查询实现（9.3.3）

### 4.1 Mapper 接口

```java
/**
 * 查询运维工单列表
 * @param task 运维工单
 * @return TaskVo集合
 */

```

### 4.2 ResultMap 关联查询

```xml
<resultMap type="taskVo" id="TaskVoResult">
    <!-- 基础字段映射 -->
    <result property="taskId" column="task_id"/>
    <result property="taskCode" column="task_code"/>
    ...
    <result property="desc" column="desc"/>
    
    <!-- 关联查询工单类型 -->
    <association property="taskType" javaType="TaskType" 
                 column="product_type_id"
                 select="com.dkd.manage.mapper.TaskTypeMapper.selectTaskTypeByTypeId"/>
</resultMap>
```

**`<association>` 关键属性：**
- `property="taskType"` — TaskVo 中的属性名
- `javaType="TaskType"` — 关联的 Java 类型
- `column="product_type_id"` — 用哪个字段作为查询条件
- `select="..."` — 调用哪个 Mapper 方法查询

### 4.3 查询语句

```xml
<select id="selectTaskVoList" resultMap="TaskVoResult">
    <include refid="selectTaskVo"/>
    <where>
        <if test="taskCode != null and taskCode != ''">and task_code = #{taskCode}</if>
        <if test="taskStatus != null">and task_status = #{taskStatus}</if>
        <if test="innerCode != null and innerCode != ''">and inner_code = #{innerCode}</if>
        ...
        <!-- 按工单类型筛选 -->
        <if test="params.isRepair != null and params.isRepair=='true'">
            and product_type_id in (1,3,4)
        </if>
        <if test="params.isRepair != null and params.isRepair=='false'">
            and product_type_id = 2
        </if>
        order by create_time desc
    </where>
</select>
```

## 七、获取运营/运维人员列表（9.4-9.5）

### 7.1 需求

根据售货机编号获取负责当前区域下的运营人员或运维人员列表。

### 7.2 实现思路

```
前端请求 /manage/emp/businessList/A1000001
    ↓
根据 innerCode 查询设备信息 → 获取 regionId
    ↓
构造查询条件：regionId + status=启用 + roleCode=运营员/维修员
    ↓
查询人员列表返回
```

### 7.3 代码实现

**Mapper 层（根据设备编号查询设备信息）：**
```java
@Select("select * from tb_vending_machine where inner_code=#{innerCode}")
VendingMachine selectVendingMachineByInnerCode(@Param("innerCode") String innerCode);
```

**Controller 层：**
```java
@GetMapping("/businessList/{innerCode}")
public AjaxResult businessList(@PathVariable("innerCode") String innerCode) {
    // 1.查询售货机信息
    VendingMachine vm = vendingMachineService.selectVendingMachineByInnerCode(innerCode);
    if (vm == null) {
        return error();
    }
    // 2.根据区域id、角色编号、员工状态查询运营人员列表
    Emp empParam = new Emp();
    empParam.setRegionId(vm.getRegionId());
    empParam.setStatus(DkdContants.EMP_STATUS_NORMAL);
    empParam.setRoleCode(DkdContants.ROLE_CODE_BUSINESS);  // 运营员
    return success(empService.selectEmpList(empParam));
}

@GetMapping("/operationList/{innerCode}")
public AjaxResult getOperationList(@PathVariable("innerCode") String innerCode) {
    // 同上，角色编码改为维修员
    empParam.setRoleCode(DkdContants.ROLE_CODE_OPERATOR);  // 维修员
    return success(empService.selectEmpList(empParam));
}
```

**常量定义：**
```java

    public static final String ROLE_CODE_BUSINESS = "1002"; // 运营员
```

### 8.4 核心业务方法 insertTaskDto（9.6.7）

**8 层逻辑：**

| 步骤 | 说明 |
|---|---|
| ① 查设备 | 根据 innerCode 查设备，不存在报错 |
| ② 校验状态 | 设备状态和工单类型要匹配（见下表） |
| ③ 查重复工单 | 同设备不能有两个进行中的同类型工单 |
| ④ 查员工 | 校验指派的员工是否存在 |
| ⑤ 校验区域 | 员工和设备必须在同一区域 |
| ⑥ 保存工单 | DTO→PO，补充工单编号、状态、地址等 |
| ⑦ 判断补货 | 只有补货工单才保存明细 |
| ⑧ 保存明细 | 批量写入 tb_task_details |

**状态校验规则：**

| 工单类型 | 设备状态要求 | 原因 |
|---|---|---|
| 投放 | 不能是"运营中" | 已运营的设备不需要投放 |
| 维修/补货/撤机 | 必须是"运营中" | 没投放的设备没法操作 |

**关键代码：**
```java
@Transactional
@Override
public int insertTaskDto(TaskDto taskDto) {
    // 1. 查询售货机
    VendingMachine vm = vendingMachineService.selectVendingMachineByInnerCode(taskDto.getInnerCode());
    if (vm == null) throw new ServiceException("设备不存在");
    
    // 2. 校验状态
    checkCreateTask(vm.getVmStatus(), taskDto.getProductTypeId());
    
    // 3. 检查重复工单
    hasTask(taskDto.getInnerCode(), taskDto.getProductTypeId());
    
    // 4. 校验员工
    Emp emp = empService.selectEmpById(taskDto.getUserId());
    if (emp == null) throw new ServiceException("员工不存在");
    
    // 5. 校验区域
    if (!emp.getRegionId().equals(vm.getRegionId())) {
        throw new ServiceException("员工区域与设备区域不一致");
    }
    
    // 6. 保存工单
    Task task = BeanUtil.copyProperties(taskDto, Task.class);
    task.setTaskStatus(DkdContants.TASK_STATUS_CREATE);
    task.setUserName(emp.getUserName());
    task.setRegionId(vm.getRegionId());
    task.setAddr(vm.getAddr());
    task.setCreateTime(DateUtils.getNowDate());
    task.setTaskCode(generateTaskCode());
    int taskResult = taskMapper.insertTask(task);
    
    // 7-8. 补货工单保存明细
    if (taskDto.getProductTypeId().equals(DkdContants.TASK_TYPE_SUPPLY)) {
        List<TaskDetailsDto> details = taskDto.getDetails();
        if (CollUtil.isEmpty(details)) throw new ServiceException("补货工单详情不能为空");
        List<TaskDetails> list = details.stream().map(dto -> {
            TaskDetails td = BeanUtil.copyProperties(dto, TaskDetails.class);
            td.setTaskId(task.getTaskId());
            return td;
        }).collect(Collectors.toList());
        taskDetailsService.batchInsertTaskDetails(list);
    }
    return taskResult;
}
```

### 8.5 Redis 生成工单编号

**格式：** `202606290001`（日期 + 4位递增序号）

```java
public String generateTaskCode() {
    String dateStr = DateUtils.getDate().replaceAll("-", "");
    String key = "dkd.task.code." + dateStr;
    if (!redisTemplate.hasKey(key)) {
        redisTemplate.opsForValue().set(key, 1, Duration.ofDays(1));
        return dateStr + "0001";
    }
    return dateStr + StrUtil.padPre(redisTemplate.opsForValue().increment(key).toString(), 4, '0');
}
```

### 9.2 Service 实现

```java
@Override
public int cancelTask(Task task) {
    // 1. 查数据库获取工单当前状态
    Task taskDb = taskMapper.selectTaskByTaskId(task.getTaskId());
    
    // 2. 校验：已取消的不能再次取消
    if (taskDb.getTaskStatus().equals(DkdContants.TASK_STATUS_CANCEL)) {
        throw new ServiceException("该工单已取消了，不能再次取消");
    }
    // 校验：已完成的不能取消
    if (taskDb.getTaskStatus().equals(DkdContants.TASK_STATUS_FINISH)) {
        throw new ServiceException("该工单已完成了，不能取消");
    }
    
    // 3. 更新状态为取消
    task.setTaskStatus(DkdContants.TASK_STATUS_CANCEL);
    task.setUpdateTime(DateUtils.getNowDate());
    return taskMapper.updateTask(task);
}
```

## 十一、踩坑记录

### 11.1 EMP_STATUS 类型不匹配

**报错：** `'setStatus(java.lang.Long)' 无法应用于 '(java.lang.Boolean)'`

**原因：** `DkdContants.EMP_STATUS_NORMAL` 定义为 `Boolean`，但 `Emp.status` 是 `Long`

**修复：** 改为 `Long` 类型
```java
// ❌ 原来
public static final Boolean EMP_STATUS_NORMAL = true;

// ✅ 修改后
public static final Long EMP_STATUS_NORMAL = 0L;
```

### 11.2 MyBatis Mapped Statements 重复定义

**报错：** `Mapped Statements collection already contains value for ...selectVendingMachineByInnerCode`

**原因：** Mapper 接口上已有 `@Select` 注解定义 SQL，XML 里又写了一份，冲突

**修复：** 删掉 XML 中的重复定义，保留注解版即可

**教训：** 同一个 Mapper 方法，注解和 XML 只能二选一！

### 11.3 Long 比较用 == 还是 equals？

```java
// ❌ 危险：Long 对象用 == 比较的是引用，不是值
if (productTypeId == DkdContants.TASK_TYPE_DEPLOY) { ... }

// ✅ 正确：用 equals 比较值
if (productTypeId.equals(DkdContants.TASK_TYPE_DEPLOY)) { ... }
```

**原因：** Java 中 `==` 比较对象引用，`equals()` 比较值。Long 在 -128~127 范围内有缓存，超出范围 `==` 会失败。

## 十二、全文总结

### 核心知识点

1. **MySQL 保留字**：`desc`、`order`、`group` 等用作字段名时需要加反引号
2. **Vo 封装**：扩展实体类，添加关联数据，用于列表展示
3. **`<association>` 关联查询**：MyBatis 一对一关联查询
4. **Lombok 版本冲突**：JDK 1.8 与高版本 Lombok 不兼容，手写 getter/setter
5. **业务接口设计**：根据设备编号 → 查区域 → 查人员
6. **DTO 设计**：接收前端数据，与数据库实体解耦
7. **批量插入**：foreach 拼接一条 SQL，比逐条插入快
8. **Redis 生成编号**：每日递增计数器，key 设置 1 天过期
9. **@Transactional**：多表操作加事务，保证数据一致性
10. **BeanUtil.copyProperties**：Hutool 属性复制工具，DTO→PO 转换
