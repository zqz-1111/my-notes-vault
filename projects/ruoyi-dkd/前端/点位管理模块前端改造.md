```html
  </el-table>
```

#### 脚本实现

```javascript
import { listNode } from "@/api/manage/node";
import { loadAllParams } from "@/api/page";

const nodeList = ref([]);
const regionInfoOpen = ref(false);

function getRegionInfo(row) {
  reset();
  getRegion(row.id).then(response => {
    form.value = response.data;
  });
  // 复用 loadAllParams，加上 regionId 筛选条件
  loadAllParams.regionId = row.id;
  listNode(loadAllParams).then(response => {
    nodeList.value = response.rows;
  });
  regionInfoOpen.value = true;
}
```

**关键点**：
- `loadAllParams` 是从 `@/api/page` 抽取的公共查询参数（pageNum=1, pageSize=10000），用于加载全量数据
- 动态添加 `regionId` 属性来筛选特定区域的点位
- 权限控制用 `manage:node:list`，因为查看详情涉及点位列表查询

## 四、方案对比

### 点位数/设备数统计方案

| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 同步存储（冗余字段） | 查询效率高 | 需要同步维护，数据可能不一致 | 数据量大、查询频繁 |
| 关联查询 | 实时准确，无需维护 | SQL 复杂，数据量大时性能低 | 数据量小、实时性要求高 |

**选择**：区域/合作商/点位数据量不大，采用关联查询方案。

## 五、踩坑避坑指南

### 逻辑坑

**坑 1：v-if 和 v-show 的选择**
- 表单字段用 `v-if` 控制显示/隐藏，因为不需要频繁切换
- `v-if` 直接销毁/重建 DOM，`v-show` 只是 CSS 隐藏
- 表单中编辑/新增字段切换用 `v-if` 更合适

**坑 2：密码回显安全问题**
- 编辑时不显示密码字段（`v-if="form.id==null"`）
- 防止密码明文回显到前端

## 六、效果验证

- 区域管理：表格显示序号、区域名称、点位数、备注说明、操作
- 合作商管理：表格显示序号、合作商名称、点位数、账号、分成比例（带%）、联系人、联系电话、操作（查看详情/修改/删除）
- 点位管理：区域和合作商用下拉选择，商圈类型用字典标签显示

## 七、全文总结

- 若依生成的代码是骨架，需要根据业务原型定制
- `v-if="form.id!=null"` 控制编辑/新增模式的字段显示
- `el-descriptions` 适合展示详情信息
- 关联查询适合数据量小的场景，同步存储适合高频查询

### 3.8 设备状态页面

设备状态是独立于设备管理的页面（vmStatus/index.vue），专注于查看设备运行状态，不需要增删改功能。

#### 运营状态 vs 设备状态

两个不同的概念，用不同方式展示：

```html
<!-- 运营状态：使用字典标签，如 未投放/运营/撤机 -->
<el-table-column label="运营状态" align="center" prop="vmStatus">

<!-- 设备状态：解析 JSON 字符串，判断设备是否正常运行 -->
<el-table-column label="设备状态" align="center" prop="vmStatus">
  <template #default="scope">
    {{ scope.row.runningStatus != null ? JSON.parse(scope.row.runningStatus).status == true ? '正常' : '异常' : '异常' }}
  </template>
</el-table-column>
```

**关键点**：
- `runningStatus` 是 JSON 字符串，存储设备硬件状态（如 `{"status": true}`）
- 用 `JSON.parse()` 解析后取 `status` 字段
- 嵌套三元运算符：先判空，再判 status 值

### 3.11 设备策略分配

在 vm/index.vue 中，点击"策略"按钮弹出策略选择对话框，为设备分配策略：

```javascript
const policyList = ref([]);
const policyOpen = ref(false);

function handleUpdatePolicy(row) {
  form.value.id = row.id;
  form.value.policyId = row.policyId;
  listPolicy(loadAllParams).then(response => {
    policyList.value = response.rows;
    policyOpen.value = true;
  });
}
```

对话框中用 el-select 下拉选择策略：

```html
<el-form-item label="策略" prop="policyId">

</el-form-item>
```

**关键点**：

### 3.13 商品管理改造

#### 价格单位转换（分 ↔ 元）

数据库存储价格单位是**分**，前端展示和输入用**元**：

```html
<!-- 表格展示：price / 100 显示元 -->
<el-table-column label="商品价格" align="center" prop="price">
  <template #default="scope">
    <el-tag>{{ scope.row.price / 100 }}元</el-tag>
  </template>
</el-table-column>

<!-- 表单输入：el-input-number 输入元 -->
<el-form-item label="商品价格" prop="price">
  <el-input-number :min="0.01" :max="999.99" :precision="2" :step="0.5" v-model="form.price" /> 元
</el-form-item>
```

脚本中做单位转换：

```javascript
// 修改时：分 → 元（回显）
function handleUpdate(row) {
  reset();
  const _skuId = row.skuId || ids.value;
  getSku(_skuId).then(response => {
    form.value = response.data;
    form.value.price = form.value.price / 100; // 分 → 元
    open.value = true;
  });
}

// 提交时：元 → 分（保存）
function submitForm() {
  proxy.$refs["skuRef"].validate(valid => {
    if (valid) {
      form.value.price = Math.round(form.value.price * 100); // 元 → 分，Math.round 避免浮点精度问题
      // ... 调用接口
    }
  });
}
```

**关键点**：
- `Math.round()` 避免 `0.1 * 100 = 10.000000000000002` 这类浮点精度问题
- el-input-number 的 `:precision="2"` 限制输入两位小数，`:step="0.5"` 步进 0.5 元

### 3.14 Excel 导入功能

#### 导入按钮


#### 导入对话框

```html
<el-dialog title="数据导入" v-model="excelOpen" width="400px" append-to-body>
  <el-upload ref="uploadRef" class="upload-demo"
    :action="uploadExcelUrl"
    :headers="headers"
    :on-success="handleUploadSuccess"
    :on-error="handleUploadError"
    :before-upload="handleBeforeUpload"
    :limit="1"
    :auto-upload="false">
    <template #trigger>
      <el-button type="primary">上传文件</el-button>
    </template>
    <el-button class="ml-3" type="success" @click="submitUpload">上传</el-button>
    <template #tip>
      <div class="el-upload__tip">上传文件仅支持，xls/xlsx格式，文件大小不得超过1M</div>
    </template>
  </el-upload>
</el-dialog>
```

#### 脚本实现

```javascript
import { getToken } from "@/utils/auth";

const excelOpen = ref(false);
const uploadExcelUrl = ref(import.meta.env.VITE_APP_BASE_API + "/manage/sku/import");
const headers = ref({ Authorization: "Bearer " + getToken() });
const uploadRef = ref({});

function handleExcelImport() {
  excelOpen.value = true;
}

function submitUpload() {
  uploadRef.value.submit(); // 手动触发上传
}

// 上传前校验
function handleBeforeUpload(file) {
  // 校验文件类型
  const fileType = ["xls", "xlsx"];
  let fileExtension = file.name.slice(file.name.lastIndexOf(".") + 1);
  const isExcel = fileType.some(type => fileExtension.indexOf(type) > -1);
  if (!isExcel) {
    proxy.$modal.msgError(`文件格式不正确, 请上传${fileType.join("/")}格式文件!`);
    return false;
  }
  // 校验文件大小
  if (file.size / 1024 / 1024 >= 1) {
    proxy.$modal.msgError("上传excel大小不能超过 1 MB!");
    return false;
  }
  proxy.$modal.loading("正在上传excel，请稍候...");
}

function handleUploadSuccess(res) {
  if (res.code === 200) {
    proxy.$modal.msgSuccess("上传excel成功");
    excelOpen.value = false;
    getList(); // 刷新列表
  } else {
    proxy.$modal.msgError(res.msg);
  }
  uploadRef.value.clearFiles();
  proxy.$modal.closeLoading();
}

function handleUploadError() {
  proxy.$modal.msgError("上传excel失败");
  uploadRef.value.clearFiles();
  proxy.$modal.closeLoading();
}
```

**关键点**：
- `auto-upload="false"` 禁止自动上传，点"上传"按钮才触发 `submitUpload()`
- `:headers` 必须携带 Authorization Token，否则后端 401
- `:limit="1"` 限制只能上传 1 个文件
- `handleBeforeUpload` 返回 `false` 阻止上传
- 上传成功后要 `clearFiles()` 清空文件列表，否则下次上传还有旧文件

#### 父组件调用子组件

```javascript
import ChannelDialog from './components/ChannelDialog.vue';

```

#### 子组件接收与通知

子组件（ChannelDialog.vue）用 `defineProps` 接收、`defineEmits` 发送事件：

```javascript
const props = defineProps({
  goodVisible: { type: Boolean, default: false },
  goodData: { type: Object, default: () => {} },
});
const emit = defineEmits(['handleCloseGood']);

// 监听 props 变化，同步到本地变量
watch(() => props.goodVisible, (val) => {
  visible.value = val;
});

// 关闭时通知父组件
const handleClose = () => {
  visible.value = false;
  emit('handleCloseGood');
};
```

#### 样式引用问题

**坑：`//` 注释 vs `<!-- -->` 注释**

```vue
// ❌ 错误：JS 注释语法放在 </script> 后面无效
// <style lang="scss" src="../index.scss"></style>

<!-- ✅ 正确：HTML 注释 -->
<!-- <style lang="scss" src="../index.scss"></style> -->
```

如果子组件不引入样式文件，但父组件引入了同一样式且没有 `scoped`，样式仍能全局生效。但这依赖父组件，不够独立。
