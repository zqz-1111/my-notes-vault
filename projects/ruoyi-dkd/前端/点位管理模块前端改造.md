### 3.1 区域管理改造

#### 基础表格改造

若依生成的表格列需要根据原型调整：

```html
<el-table v-loading="loading" :data="regionList" @selection-change="handleSelectionChange">
  <el-table-column type="selection" width="55" align="center" />
  <!-- type="index" 自动生成递增序号 -->
  <el-table-column label="序号" type="index" width="50" align="center" prop="id" />
  <el-table-column label="区域名称" align="center" prop="regionName" />
  <!-- 点位数：后端通过关联查询返回 nodeCount 字段 -->
  <el-table-column label="点位数" align="center" prop="nodeCount" />
  <el-table-column label="备注说明" align="center" prop="remark" />
  <el-table-column label="操作" align="center" class-name="small-padding fixed-width">
    <template #default="scope">
      <el-button link type="primary" @click="handleUpdate(scope.row)" v-hasPermi="['manage:region:edit']">修改</el-button>
      <el-button link type="primary" @click="handleDelete(scope.row)" v-hasPermi="['manage:region:remove']">删除</el-button>
    </template>
  </el-table-column>
</el-table>
```

**关键点**：
- `type="index"` 让序号列自动递增，不需要后端返回
- `prop="nodeCount"` 对应后端 RegionVo 的 nodeCount 字段（关联查询统计点位数）
- 操作按钮去掉了 icon，保持简洁

### 3.3 区域管理 — 查看详情功能

区域列表操作列新增"查看详情"按钮，点击弹出对话框显示区域名称和该区域下的点位列表。

#### 按钮与对话框

```html
<el-button link type="primary" @click="getRegionInfo(scope.row)" v-hasPermi="['manage:node:list']">查看详情</el-button>

<el-dialog title="区域详情" v-model="regionInfoOpen" width="500px" append-to-body>
  <el-form-item label="区域名称" prop="regionName">
    <el-input v-model="form.regionName" disabled />
  </el-form-item>
  <label>包含点位：</label>
  <el-table :data="nodeList">
    <el-table-column label="序号" type="index" width="50" align="center" />
    <el-table-column label="点位名称" align="center" prop="nodeName" />
    <el-table-column label="设备数量" align="center" prop="vmCount" />
  </el-table>
</el-dialog>
```

#### 脚本实现

```javascript
import { listNode } from "@/api/manage/node";
import { loadAllParams } from "@/api/page";

const nodeList = ref([]);
const regionInfoOpen = ref(false);

function getRegionInfo(row) {
  reset();
  getRegion(row.id).then(response => {
    form.value = response.data;
  });
  // 复用 loadAllParams，加上 regionId 筛选条件
  loadAllParams.regionId = row.id;
  listNode(loadAllParams).then(response => {
    nodeList.value = response.rows;
  });
  regionInfoOpen.value = true;
}
```

**关键点**：
- `loadAllParams` 是从 `@/api/page` 抽取的公共查询参数（pageNum=1, pageSize=10000），用于加载全量数据
- 动态添加 `regionId` 属性来筛选特定区域的点位
- 权限控制用 `manage:node:list`，因为查看详情涉及点位列表查询

## 四、方案对比

### 点位数/设备数统计方案

| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 同步存储（冗余字段） | 查询效率高 | 需要同步维护，数据可能不一致 | 数据量大、查询频繁 |
| 关联查询 | 实时准确，无需维护 | SQL 复杂，数据量大时性能低 | 数据量小、实时性要求高 |

**选择**：区域/合作商/点位数据量不大，采用关联查询方案。

## 五、踩坑避坑指南

### 逻辑坑

**坑 1：v-if 和 v-show 的选择**
- 表单字段用 `v-if` 控制显示/隐藏，因为不需要频繁切换
- `v-if` 直接销毁/重建 DOM，`v-show` 只是 CSS 隐藏
- 表单中编辑/新增字段切换用 `v-if` 更合适

**坑 2：密码回显安全问题**
- 编辑时不显示密码字段（`v-if="form.id==null"`）
- 防止密码明文回显到前端

## 六、效果验证

- 区域管理：表格显示序号、区域名称、点位数、备注说明、操作
- 合作商管理：表格显示序号、合作商名称、点位数、账号、分成比例（带%）、联系人、联系电话、操作（查看详情/修改/删除）
- 点位管理：区域和合作商用下拉选择，商圈类型用字典标签显示

## 七、全文总结

- 若依生成的代码是骨架，需要根据业务原型定制
- `v-if="form.id!=null"` 控制编辑/新增模式的字段显示
- `el-descriptions` 适合展示详情信息
- 关联查询适合数据量小的场景，同步存储适合高频查询
